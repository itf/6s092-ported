# Introduction to Networking

A **network** is a collection of nodes connected by links, designed to forward packets from source to destination. The fundamental challenge is **routing**: how does a packet find its way from source to destination in a large, dynamic network where links can fail and new nodes join?

The internet is organized in **layers**: the link layer handles communication on a single physical link; the network layer (IP) handles routing across multiple hops; the transport layer (TCP/UDP) handles end-to-end reliability and demultiplexing; the application layer handles application-specific protocols. Each layer provides a service to the layer above via a well-defined interface, hiding implementation details below.

**Packet switching** (used by the internet) sends data in discrete chunks (packets) that each independently find their way to the destination. In **circuit switching** (used by the telephone system), a dedicated path is reserved before data is sent. Packet switching allows better statistical multiplexing of shared links but introduces variable latency and the possibility of packet loss.

## Packet Switching vs. Circuit Switching

In packet switching, each router independently decides where to forward each packet based on its destination address and a routing table. Packets from different flows share links—a bursty flow can use idle capacity from quiet flows. The downside is **head-of-line blocking** and variable queueing delays.

<checkyourself>
Why does packet switching provide better utilization of shared links than circuit switching for bursty traffic?

<showhide>
Circuit switching reserves a fixed bandwidth slice for each connection for its entire duration, even when the connection is idle. Bursty traffic (e.g., web browsing) is idle most of the time, wasting reserved capacity. Packet switching allows multiple flows to share the link: when one flow is idle, other flows can use its capacity. This **statistical multiplexing** means average utilization is much higher, but at the cost of variable delay when multiple flows are simultaneously active.

## Layering and Encapsulation

Each layer adds its own header (**encapsulation**). An application sends data; the transport layer adds a TCP header; the network layer adds an IP header; the link layer adds a frame header. At the destination, each layer strips its header and passes the payload up. This allows layers to be independently designed and upgraded.

What is the difference between a switch (layer-2) and a router (layer-3), and what header does each use for forwarding decisions?

<showhide>
A **switch** operates at layer 2 (link layer) and forwards frames based on **MAC addresses**. It learns MAC-to-port mappings by observing traffic and uses them to forward unicast frames; unknown destinations are flooded. A **router** operates at layer 3 (network layer) and forwards packets based on **IP addresses** using a routing table. Routers cross network boundaries and separate broadcast domains; switches connect hosts within the same network.
