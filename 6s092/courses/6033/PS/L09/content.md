# Routing

Routing algorithms compute the paths packets take through a network. The two classical algorithms are **link-state routing** and **distance-vector routing**. Both find shortest paths (by some metric like hop count or link cost) but differ in what information each router maintains and how they exchange updates.

In **link-state routing** (e.g., OSPF), each router floods its local link state (its neighbors and link costs) to all routers in the network. Every router builds a complete map of the network topology and runs Dijkstra's algorithm locally to compute shortest paths to all destinations. Updates are fast to converge but require $O(N)$ state per router and $O(N^2)$ messages for a full update.

In **distance-vector routing** (e.g., RIP), each router only knows its neighbors and their advertised distances to destinations. Routers exchange distance vectors with neighbors and update their tables using the Bellman-Ford equation: $d(u,v) = \min_w [c(u,w) + d(w,v)]$. This requires less per-router state but converges more slowly and suffers from the **count-to-infinity** problem when links fail.

## Link-State Routing

Every router floods its link-state advertisement (LSA) using reliable flooding: each router forwards each LSA exactly once (using sequence numbers to detect duplicates). After flooding converges, all routers have an identical copy of the network topology. Each then independently runs Dijkstra's to build its forwarding table.

<checkyourself>
Why does link-state routing converge faster than distance-vector routing after a link failure?

<showhide>
In link-state routing, the failure is immediately flooded to all routers. Within one flooding round, every router has the updated topology and recomputes paths independently. In distance-vector, a failure propagates iteratively: each router updates its table based on neighbors' advertisements, then neighbors update based on those, and so on. The count-to-infinity problem can cause this to take many rounds—potentially counting up to a large metric before the failure is recognized.

## Distance-Vector and Bellman-Ford

The Bellman-Ford update rule: router $u$ computes distance to destination $v$ as $d(u,v) = \min_{w \in \text{neighbors}(u)} [c(u,w) + d(w,v)]$. Each iteration propagates knowledge one more hop. In a network with $N$ nodes, at most $N-1$ iterations are needed for convergence.

What is split horizon and how does it mitigate the count-to-infinity problem?

<showhide>
Split horizon: a router does not advertise a route back to the neighbor from whom it learned that route. If A learned the route to X from B, A will not include X in the distance vector it sends to B. This prevents B from thinking it can reach X via A when A's path to X is through B—breaking the routing loop. Split horizon with poison reverse goes further: instead of omitting the route, A advertises it to B with metric infinity (poisoning the route), which causes B to immediately invalidate its path through A.
</showhide>
