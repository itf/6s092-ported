# Network Security: DDoS and Intrusion Detection

**Distributed Denial of Service (DDoS)** attacks aim to make a service unavailable by overwhelming it with traffic. Unlike most attacks, DDoS doesn't require compromising the target—it exploits the internet's open architecture, where anyone can send packets to anyone. The attacker controls a **botnet** (thousands of compromised hosts) that floods the target with traffic.

DDoS defense is fundamentally hard because: (1) the internet was designed for openness, not security; (2) attack traffic looks similar to legitimate traffic; (3) filtering at the target is too late—the link to the target is already saturated. Effective defense requires filtering closer to the sources (**ingress filtering**) or using traffic scrubbing centers that sit upstream.

**Intrusion Detection Systems (IDS)** monitor network traffic or system behavior for signs of attacks. **Signature-based IDS** match traffic against known attack patterns (fast, low false positives, but fails against novel attacks). **Anomaly-based IDS** learn normal behavior and flag deviations (detects novel attacks but has more false positives).

## DDoS Attack Mechanics

**Amplification attacks** exploit protocols that send large responses to small requests. An attacker sends a request spoofing the victim's source IP; the server sends a large response to the victim. DNS amplification: a 60-byte query can elicit a 3000-byte response—50× amplification. NTP monlist: up to 4000× amplification. The attacker gets massive bandwidth directed at the victim with little bandwidth of their own.

<checkyourself>
Why is IP address spoofing possible in a DDoS amplification attack, and what network-level mechanism could prevent it?

<showhide>
IP is a connectionless protocol with no source address verification. A sender can put any IP address in the source field of a packet—there is no enforcement. In an amplification attack, the attacker puts the victim's IP as the source address; the amplifier (DNS server, NTP server) sends its large response to the victim. **Ingress filtering (BCP38)**: ISPs should drop packets at their edge that have source addresses not belonging to their assigned address blocks. If all ISPs enforced this, spoofed packets would be dropped before reaching amplifiers. Unfortunately, BCP38 deployment is incomplete because individual ISPs don't bear the cost of the attacks their customers enable.

## Intrusion Detection

A **firewall** is the first line of defense: a stateful packet filter that allows or blocks traffic based on rules (IP address, port, protocol, connection state). Firewalls enforce a **perimeter model**: trusted inside, untrusted outside. But perimeters are insufficient—threats inside the perimeter (insider attacks, compromised internal hosts) evade the firewall.

What is the difference between a stateless and stateful firewall, and why does stateful inspection provide better security?

<showhide>
A **stateless firewall** filters each packet independently based on header fields (src/dst IP, port, protocol). It cannot distinguish between a TCP SYN (initiating a connection) and a TCP ACK (part of an established connection) that happens to match the header filter—an attacker can craft a packet that looks like a response to allow it through. A **stateful firewall** tracks connection state: it knows which TCP connections are established and allows inbound packets only for connections initiated from the inside. This blocks unsolicited inbound packets that would bypass a stateless filter, while still allowing responses to legitimate outbound connections.
