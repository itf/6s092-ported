# Introduction, Linear Classification, Perceptron Update Rule

Machine learning is concerned with building systems that learn from data. In **supervised learning**, we are given labeled training examples $(x^{(i)}, y^{(i)})$ and want to learn a function $f$ that predicts $y$ from $x$. For **binary classification**, $y \in \{-1, +1\}$. A **linear classifier** uses a weight vector $w$ and bias $b$ to predict $\hat{y} = \text{sign}(w \cdot x + b)$. The **perceptron** is a classic online algorithm that updates its weights whenever it makes a mistake.

## Linear Classification

A linear classifier partitions the input space with a hyperplane. Points on one side are labeled $+1$, points on the other $-1$. The decision boundary is $\{x : w \cdot x + b = 0\}$.

<checkyourself>
If $w = (1, 2)$ and $b = -3$, what is the predicted label for $x = (2, 1)$?

<showhide>
$w \cdot x + b = 1 \cdot 2 + 2 \cdot 1 - 3 = 1 > 0$, so the predicted label is $+1$.
</showhide>
</checkyourself>

## The Perceptron Algorithm

The perceptron update rule: if a training example $(x^{(i)}, y^{(i)})$ is misclassified (i.e., $y^{(i)}(w \cdot x^{(i)}) \leq 0$), update:
$$w \leftarrow w + y^{(i)} x^{(i)}$$

The algorithm cycles through the data until no mistakes are made (if the data is linearly separable).

<checkyourself>
Why is it important to include a bias term $b$ in a linear classifier, rather than just using $w \cdot x$?

<showhide>
Without a bias term, the decision boundary must pass through the origin. Including $b$ allows the hyperplane to be placed anywhere in the input space, making the classifier strictly more expressive.
