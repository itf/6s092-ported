# Kernel Regression and Kernels

**Kernel regression** extends regression to non-linear settings using the kernel trick. In the **representer theorem** framework, the optimal solution $w^*$ in the reproducing kernel Hilbert space (RKHS) lies in the span of the training feature vectors $\phi(x^{(i)})$, so we can write $w^* = \sum_i \alpha_i \phi(x^{(i)})$ and express everything via the kernel matrix.

## Representer Theorem

For regularized objectives of the form $\min_w \sum_i L(y^{(i)}, w \cdot \phi(x^{(i)})) + \lambda \|w\|^2$, the optimal $w$ can be written as:
$$w^* = \sum_{i=1}^n \alpha_i \phi(x^{(i)})$$

Predictions: $w^* \cdot \phi(x) = \sum_i \alpha_i K(x^{(i)}, x)$.

<checkyourself>
Why does the representer theorem allow kernel methods to work efficiently even when $\phi(x)$ is infinite-dimensional?

<showhide>
Because we only need the $\alpha_i$ coefficients (one per training example), not $w$ itself. Predictions and training both require only kernel evaluations $K(x^{(i)}, x^{(j)})$, which are computable even when $\phi$ is infinite-dimensional.
</showhide>
</checkyourself>

## Kernel Regression (Nadaraya-Watson)

A non-parametric approach: predict at $x$ as a weighted average of training labels, where weights depend on similarity via the kernel:
$$\hat{y}(x) = \frac{\sum_i K(x, x^{(i)}) y^{(i)}}{\sum_i K(x, x^{(i)})}$$

The bandwidth (e.g., $\sigma$ in the RBF kernel) controls the smoothness of the fit.

What is the role of $\lambda$ (the regularization parameter) in kernelized ridge regression?

<showhide>
$\lambda$ controls the tradeoff between data fit and model complexity. Larger $\lambda$ shrinks the $\alpha$ coefficients toward zero, producing a smoother function. Smaller $\lambda$ allows the model to fit the training data more closely, potentially overfitting.
