# Classification Errors, Regularization, Logistic Regression

Real datasets are rarely perfectly separable. The **soft-margin SVM** introduces slack variables to allow some misclassification, controlled by a regularization parameter $C$. **Logistic regression** is a probabilistic approach to binary classification that models the posterior probability $P(y=1 \mid x)$ using the sigmoid function, and is trained by maximizing the log-likelihood with optional regularization.

## Soft-Margin SVM

Allow misclassification by introducing slack $\xi_i \geq 0$:
$$\min_{w,b,\xi} \frac{1}{2}\|w\|^2 + C \sum_i \xi_i \quad \text{s.t.} \quad y^{(i)}(w \cdot x^{(i)} + b) \geq 1 - \xi_i, \; \xi_i \geq 0$$

Large $C$ penalizes errors heavily (approaches hard-margin); small $C$ allows more errors.

<checkyourself>
What happens as $C \to \infty$ in the soft-margin SVM?

<showhide>
As $C \to \infty$, any margin violation is infinitely penalized, so the solution approaches the hard-margin SVM (if the data is separable). If the data is not separable, larger $C$ means fewer training errors but potentially worse generalization.
</showhide>
</checkyourself>

## Logistic Regression

Logistic regression models:
$$P(y=1 \mid x) = \sigma(w \cdot x + b) = \frac{1}{1 + e^{-(w \cdot x + b)}}$$

The parameters are found by maximizing the log-likelihood:
$$\ell(w, b) = \sum_i \log P(y^{(i)} \mid x^{(i)})$$

Regularization adds a penalty $-\frac{\lambda}{2}\|w\|^2$ to prevent overfitting.

<showhide>
