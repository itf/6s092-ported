# NP-completeness

A problem $L$ is **NP-hard** if every problem in NP reduces to $L$ in polynomial time. It is **NP-complete** if it is both NP-hard and in NP. NP-complete problems are the hardest in NP: solving any one in polynomial time would solve all of NP (and thus imply P = NP). The Cook-Levin theorem (1971) showed that SAT is NP-complete. The key technique is *polynomial-time many-one reduction*: $A \leq_P B$ means there is a polytime computable function $f$ such that $x \in A \iff f(x) \in B$.

This lecture shows how to reduce SAT to **3SAT** (at most 3 literals per clause). The argument converts any Boolean formula to a circuit and then to a 3CNF formula by introducing auxiliary variables for each gate. Because SAT reduces to 3SAT, and 3SAT reduces to many natural problems (clique, vertex cover, Hamiltonian path, ...), all of these are NP-complete.

## Cook-Levin and the First NP-Complete Problem

To show SAT is NP-complete: (1) SAT is in NP (a satisfying assignment is the certificate); (2) every NP problem reduces to SAT. The reduction encodes the computation of the NP verifier as a circuit, and asks whether there is an input (the certificate) that makes the circuit output 1. This is an instance of Circuit-SAT, which is then converted to CNF-SAT.

## SAT Reduces to 3SAT

Given an arbitrary Boolean formula, first convert to a circuit (linear time). Then, for each gate with output $z$ and inputs $a, b$, introduce a clause set enforcing the gate's truth table: e.g., an AND gate $z = a \wedge b$ gives clauses $(\neg a \vee \neg b \vee z) \wedge (a \vee \neg z) \wedge (b \vee \neg z)$. Each gate produces $O(1)$ clauses of width $\leq 3$. The resulting 3CNF formula is satisfiable iff the original circuit is satisfiable.

<checkyourself>
Why is 2SAT in P while 3SAT is NP-complete? What changes when you go from 2 to 3 literals per clause?

<showhide>
2SAT can be solved in linear time via the implication graph: each clause $(\ell_1 \vee \ell_2)$ gives implications $\neg \ell_1 \Rightarrow \ell_2$ and $\neg \ell_2 \Rightarrow \ell_1$. The formula is unsatisfiable iff some variable $x$ and its negation $\neg x$ are in the same strongly connected component. With 3 literals per clause, this implication structure breaks down and the problem encodes general search.
