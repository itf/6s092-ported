# Circuits and Finite Automata

This lecture introduces two fundamental models of computation: Boolean circuits and finite automata. A **Boolean circuit** is a directed acyclic graph whose nodes are logic gates (AND, OR, NOT) with designated input and output bits. A circuit of size $s$ can compute any Boolean function $f: \{0,1\}^n \to \{0,1\}$ that requires at most $s$ gates. A key limitation is that a circuit must be *hardwired* for a fixed input size — you need a different circuit for each $n$.

**Finite automata** (DFAs and NFAs) process strings of arbitrary length by reading one symbol at a time and transitioning between a finite set of states. A DFA accepts a string if it ends in an accepting state. The class of languages recognizable by DFAs is exactly the *regular languages*. Unlike circuits, DFAs handle arbitrary-length inputs, but their memory is fixed — they cannot count arbitrarily.

## Boolean Circuits

Every Boolean function $f: \{0,1\}^n \to \{0,1\}$ can be computed by some circuit of size at most $O(2^n)$ (just enumerate all $2^n$ inputs). Claude Shannon showed by a counting argument that *most* functions require circuits of size $\Omega(2^n / n)$: there are $2^{2^n}$ Boolean functions but only $s^{O(s)}$ circuits of size $s$, so for small $s$ most functions are not covered.

## Finite Automata

A DFA is a 5-tuple $(Q, \Sigma, \delta, q_0, F)$ where $Q$ is a finite set of states, $\Sigma$ is the input alphabet, $\delta: Q \times \Sigma \to Q$ is the transition function, $q_0 \in Q$ is the start state, and $F \subseteq Q$ is the set of accepting states. The key limitation: a DFA with $k$ states cannot distinguish strings that lead to the same state after $k$ steps. The *pumping lemma* formalizes this: for any regular language $L$, there exists a length $p$ such that any string of length $\geq p$ in $L$ can be "pumped" (a middle section repeated) and remain in $L$.
