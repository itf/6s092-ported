# P and NP

The class **NP** (nondeterministic polynomial time) consists of decision problems for which a "yes" answer has a short, efficiently verifiable *certificate* (witness). Formally, $L \in NP$ if there exists a polynomial-time verifier $V$ and a polynomial $p$ such that $x \in L \iff \exists w \in \{0,1\}^{p(|x|)},\, V(x, w) = 1$. Examples: SAT (the certificate is a satisfying assignment), 3-coloring (the certificate is the coloring), clique (the certificate is the clique itself).

The P vs NP question asks: is efficient *verification* equivalent to efficient *solving*? This is arguably the most important open problem in mathematics. The general belief is $P \neq NP$, but no proof is known. The "DUH" problem — given a Turing machine and $n, k$, is there an input of size $n$ accepted in fewer than $n^k$ steps? — is "obviously" NP-complete.

## NP and Certificates

NP is closed under polynomial-time many-one reductions. The class **coNP** contains problems whose "no" instances have short certificates. For example, *unsatisfiability* of a Boolean formula is in coNP (a "no" certificate for SAT). Whether NP = coNP is another major open question.

## The Landscape of Problems

Problems believed to be in NP but not in P include: 3-SAT, 3-coloring, max clique, traveling salesman (decision version), bin packing, and the "Theorem Problem" (is there a proof of $\phi$ with at most $n$ symbols in ZF?). All of these share the property that given a proposed solution, you can verify it in polynomial time — but finding a solution seems to require exponential search.

<showhide>
Modern cryptography relies on one-way functions: functions easy to compute but hard to invert. If $P = NP$, then inverting any function whose output is verifiable in polynomial time would also be solvable in polynomial time. This would break RSA (factoring would be easy), break discrete-log-based cryptography, and likely destroy all current public-key cryptosystems.
