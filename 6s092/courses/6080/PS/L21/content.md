# Learning, Chomsky, RSA, Quantum

This lecture weaves together several threads. First: the connection between **cryptography and learning hardness**. A concept class $C$ is hard to PAC-learn if learning it would break RSA. The construction: take $c_N$ to be the concept for the RSA modulus $N$, where a sample $(y, i)$ is labeled 1 iff the $i$-th bit of $y^{1/3} \bmod N$ is 1. A PAC learner for this class could recover bits of $x$ from $x^3 \bmod N$, breaking RSA.

Third: a preview of quantum computing — the next lectures will show that quantum computers can break RSA by factoring in polynomial time (Shor's algorithm).

A learning algorithm, given many (ciphertext, plaintext-bit) pairs as training data, must generalize — but this is exactly the ability to invert RSA.

## Chomsky's Language Hierarchy

The Chomsky hierarchy classifies formal languages by the type of grammar that generates them:
- **Type 3** (Regular): recognized by DFAs
- **Type 2** (Context-free): recognized by pushdown automata, includes most programming language syntax
- **Type 1** (Context-sensitive): recognized by linear-bounded automata
- **Type 0** (Recursively enumerable): recognized by Turing machines

Each class is strictly contained in the next. Context-free languages include $\{0^n 1^n\}$ (not regular) but not $\{0^n 1^n 2^n\}$ (context-sensitive).

<checkyourself>
Where does the language $\{0^n 1^n 2^n : n \geq 0\}$ sit in the Chomsky hierarchy, and why isn't it context-free?

<showhide>
It is context-sensitive (Type 1) but not context-free (Type 2). The pumping lemma for CFLs says any sufficiently long string in a CFL has a "pumpable" middle section. For $0^n 1^n 2^n$, any pumping would unbalance the counts of 0s, 1s, and 2s. Intuitively, a pushdown automaton (recognizing CFLs) has only one stack, which is not enough to simultaneously count all three symbols.
