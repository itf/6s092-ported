# Private-key Cryptography

Private-key (symmetric) cryptography uses a shared secret key known to both sender and receiver. The **Caesar cipher** (shift by 3) is trivially breakable. The **Vernam one-time pad** (OTP) achieves *information-theoretic* security: if the key is truly random and as long as the message, no adversary — even with unlimited computation — can recover the message. The key insight: the ciphertext $c = m \oplus k$ is uniformly distributed regardless of $m$, so it leaks nothing about $m$.

The OTP's fatal flaw: the key must be as long as the message, and keys cannot be reused. If a key $k$ is used twice, an eavesdropper who observes $c_1 = m_1 \oplus k$ and $c_2 = m_2 \oplus k$ can compute $c_1 \oplus c_2 = m_1 \oplus m_2$, leaking information about both messages. Cold War Soviet spies were actually caught this way.

## One-Time Pad Security

Shannon proved that the one-time pad achieves *perfect secrecy*: for every distribution over messages and every ciphertext $c$, $\Pr[M = m \mid C = c] = \Pr[M = m]$. The ciphertext is statistically independent of the message. Shannon also proved a converse: any perfectly secret encryption scheme requires a key at least as long as the message.

## Pseudorandom Generators for Encryption

Since the one-time pad requires keys as long as messages, practical private-key cryptography uses **stream ciphers**: take a short random seed $s$ (the secret key), expand it to a long pseudorandom string $G(s)$, and use $G(s)$ as a one-time pad. Security relies on $G$ being a PRG: no polynomial-time adversary can distinguish $G(s)$ from truly random, so no polynomial-time adversary can break the scheme.

What is the difference between information-theoretic security (like the OTP) and computational security (like a stream cipher based on a PRG)?

<showhide>
Information-theoretic security holds against adversaries with *unlimited* computational power. The OTP is secure even if the adversary has an exponentially fast computer. Computational security holds only against *polynomial-time* adversaries. A stream cipher can be broken if the adversary can run in superpolynomial time — specifically, if they can invert the PRG. The OTP is provably unbreakable; the stream cipher is secure only under the assumption that the PRG is hard to break.
