# Probabilistic Complexity Classes

This lecture formalizes the role of randomness in efficient computation. The class **BPP** (Bounded-error Probabilistic Polynomial time) contains problems solvable by a polynomial-time randomized algorithm with error probability $\leq 1/3$. The specific threshold $1/3$ is arbitrary: by *amplification* (running the algorithm $k$ times and taking the majority), the error probability drops exponentially in $k$. The Chernoff bound makes this precise.

Key probabilistic complexity classes:
- **P**: deterministic polynomial time
- **ZPP**: zero-error expected polynomial time ($= RP \cap coRP$)
- **RP**: one-sided error (no false positives)
- **coRP**: complement of RP (no false negatives)
- **BPP**: two-sided error, bounded away from $1/2$

It is conjectured that $P = BPP$ (all randomness can be derandomized efficiently), but this is not proven.

## The BPP Definition and Amplification

$L \in BPP$ iff $\exists$ poly-time $M(x, r)$: if $x \in L$, $\Pr_r[M \text{ accepts}] \geq 2/3$; if $x \notin L$, $\Pr_r[M \text{ accepts}] \leq 1/3$. By amplification: run $M$ independently $t$ times, accept if the majority accepts. The Chernoff bound says that if each trial is wrong with probability $\leq 1/3$, then after $t = O(\log(1/\delta))$ trials, the majority is wrong with probability $\leq \delta$.

## The Chernoff Bound

For $n$ independent coin flips $X_1, \ldots, X_n \in \{0,1\}$ with $E[X_i] = p$, let $X = \sum X_i$. The Chernoff bound says: $\Pr[|X - np| > \epsilon n] \leq 2e^{-2\epsilon^2 n}$. This exponential tail means that the empirical frequency concentrates extremely tightly around $p$ for large $n$. The proof uses the moment generating function $E[e^{tX}] = (pe^t + (1-p))^n$.

$RP$ has no false positives (if $x \in L$, it accepts with probability $\geq 1/2$; if $x \notin L$, it always rejects). $coRP$ has no false negatives (if $x \notin L$, it rejects with probability $\geq 1/2$; if $x \in L$, it always accepts). An algorithm in $RP \cap coRP$ has neither false positives nor false negatives — it is always correct when it gives an answer — but it may say "I don't know" with probability $\leq 1/2$. Running it again gives a definitive answer in expected $O(1)$ trials, giving expected polynomial time (ZPP).
