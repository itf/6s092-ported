# NP-completeness in Practice

Encountering an NP-complete problem in practice does not mean giving up. There are at least seven strategies for coping: (1) **brute force** (fine for small $n$); (2) **structure exploitation** (your instance may have special form); (3) **special casing** (solve a tractable subproblem containing your instance, e.g., 2SAT instead of 3SAT); (4) **parameterized algorithms** (if a parameter $k$ besides $n$ is small, e.g., clique of size $k$ in $O(n^k)$ time); (5) **approximation algorithms** (find a solution within a known factor of optimal); (6) **heuristics** like simulated annealing (no guarantee, but often work well); (7) **randomized search**.

Another important topic is **factoring** as a problem that is believed to be outside P but is also not NP-complete. Factoring is in coNP ∩ NP (the primality certificate from AKS), and it is conjectured to be in a "middle zone" between P and NP-complete. This matters greatly for cryptography.

## Approximation Algorithms

For optimization versions of NP-hard problems, we often seek *approximation algorithms* — algorithms that run in polynomial time and return a solution within a factor $\alpha$ of optimal. For example, vertex cover has a 2-approximation (take both endpoints of each edge in a maximal matching). The **PCP theorem** (1992) shows that some problems have no polynomial-time approximation within any constant factor unless P = NP.

## Parameterized Complexity

Parameterized complexity studies algorithms that are efficient when some parameter $k$ of the input is small. An algorithm running in $f(k) \cdot p(n)$ time (where $f$ can be anything and $p$ is polynomial) is called *fixed-parameter tractable* (FPT). For example, vertex cover has an FPT algorithm: either include a vertex or all its neighbors, branching on each edge. This gives $O(2^k \cdot n)$ time — exponential in $k$ but polynomial in $n$.

<showhide>
A parameterized algorithm provides a *guarantee*: if the parameter $k$ is small, the algorithm is provably fast. A heuristic like simulated annealing may work well on average but offers no worst-case guarantees. For safety-critical applications or when you need an exact solution with a certificate, the parameterized algorithm is preferable.
