# Turing Machines

The Turing machine overcomes the limitations of both circuits (fixed input size) and finite automata (fixed memory) by combining a read/write tape with the ability to move in both directions and halt at any time. At each step, a Turing machine faces three questions: (1) change state? (2) write to the tape? (3) move left, right, or halt? The answers depend only on the current state and the tape symbol being read. The unlimited tape provides unlimited memory, making Turing machines the right model for capturing all of computation.

The most celebrated result is the existence of a **Universal Turing Machine** (UTM): a single Turing machine $U$ that, given an encoding $\langle M, x \rangle$ of any other machine $M$ and input $x$, simulates $M$ on $x$. This is Turing's "existence of the software industry" lemma from his 1936 paper. Without universality, you would need new hardware for every new problem; universality is what makes software possible.

## Turing Machine Formalism

A Turing machine is a tuple $(Q, \Gamma, \Sigma, \delta, q_0, q_{\text{accept}}, q_{\text{reject}})$ where $Q$ is the set of states, $\Gamma$ is the tape alphabet (including blank symbol $\sqcup$), $\Sigma \subseteq \Gamma \setminus \{\sqcup\}$ is the input alphabet, and $\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R, H\}$ is the transition function. The palindrome problem can be solved in $O(n^2)$ time by a Turing machine using a back-and-forth marking strategy; interestingly, it is known that $\Omega(n^2)$ steps are necessary on a single-tape machine.

## Universal Turing Machines

A UTM encodes the description of a target machine $M$ as a string on its tape, then simulates $M$ step by step. This shows that there is a single fixed program (the UTM) that can perform any computation — the essence of general-purpose computing. The **Church-Turing thesis** says that Turing machines capture the intuitive notion of computation: anything reasonably called a computer can be simulated by a Turing machine with at most polynomial overhead.

The UTM reads $M$'s description to determine its transitions, then simulates each step of $M$ by updating a representation of $M$'s current state and tape content. The UTM accepts if and only if $M$ accepts; it rejects if $M$ rejects; it loops if $M$ loops.
