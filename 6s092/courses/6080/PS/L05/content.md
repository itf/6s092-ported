# Reducibility and Gödel

The **halting problem** — given a Turing machine $M$ and input $x$, does $M$ halt on $x$? — is undecidable. The classic proof is by diagonalization: suppose a machine $H$ could solve the halting problem. Build a machine $D$ that, on input $\langle M \rangle$, runs $H(\langle M \rangle, \langle M \rangle)$ and does the opposite. Then $D(\langle D \rangle)$ leads to a contradiction. There are more computational problems (uncountably many) than there are programs (countably many), so most problems are uncomputable.

**Oracles** (Turing's 1939 invention) are hypothetical devices that answer a specific question for free. If machine $M$ has access to oracle $A$, we write $M^A$. Oracles let us define a *reducibility* relation: $A \leq_T B$ means "given oracle $B$, you can solve $A$." This creates a hierarchy of difficulty (Turing degrees) and lets us prove relative hardness results independent of whether either problem is actually decidable.

## The Halting Problem

The halting problem $\text{HALT} = \{\langle M, x \rangle : M \text{ halts on } x\}$ is *recursively enumerable* (you can enumerate halting computations) but not *recursive* (decidable). The complement — "does $M$ run forever?" — is not even recursively enumerable. The number of Turing machines is countable (each is a finite string over a finite alphabet), while the number of languages over $\{0,1\}^*$ is uncountable (by the same argument as for real numbers), so most languages are undecidable.

## Oracles and Turing Degrees

An oracle $A$ is a set $A \subseteq \{0,1\}^*$; a machine $M^A$ can query it in a single step. The *Turing degree* of a problem is its equivalence class under $\leq_T$ (both $A \leq_T B$ and $B \leq_T A$). HALT has its own degree; there exist problems harder than HALT (e.g., "does $M^{\text{HALT}}$ halt?"), and by a theorem of Friedberg and Muchnik, there exist degrees strictly between computable and HALT.
