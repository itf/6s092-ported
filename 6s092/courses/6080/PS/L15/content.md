# Derandomization / Cryptography

Two remarkable derandomization results suggest that randomness may be less essential than it appears. (1) The **AKS primality test** (Agrawal-Kayal-Saxena, 2002): given $N$, decide in $O((\log N)^6)$ deterministic time whether $N$ is prime. Before AKS, the best known deterministic test required assuming GRH. (2) **Reingold's algorithm** (2005): undirected $s$-$t$ connectivity is in deterministic LOGSPACE (L), replacing the previously known randomized LOGSPACE algorithm.

These results, combined with other evidence, support the conjecture $P = BPP$: every randomized polynomial-time algorithm can be derandomized. The key tool would be a **pseudorandom generator** (PRG): a deterministic function that stretches a short seed into a long string indistinguishable from truly random by any efficient adversary.

## Pseudorandom Generators

A PRG is a polytime function $G: \{0,1\}^n \to \{0,1\}^{n+1}$ such that no polytime algorithm $A$ can distinguish $G(x)$ (for random $x \in \{0,1\}^n$) from a truly random string in $\{0,1\}^{n+1}$. If a PRG exists, you can derandomize BPP by replacing the random tape with $G$'s output over all seeds and taking the majority — this requires only $2^n$ deterministic trials for a seed of length $n$.

## From PRGs to Derandomization

Given a PRG $G: \{0,1\}^n \to \{0,1\}^{p(n)}$ and a BPP algorithm $A$ that uses $p(n)$ random bits, we can derandomize: run $A$ with $G(s)$ as its random tape for each of the $2^n$ seeds $s$, and output the majority answer. If $G$ is secure, this works because any BPP algorithm is a polynomial-time distinguisher, so it can't tell $G(s)$ from truly random. The result is a deterministic algorithm running in $2^n \cdot \text{poly}(n)$ time.

<checkyourself>
Why does the existence of a PRG not immediately prove $P = BPP$?

<showhide>
The derandomization using a PRG with seed length $n$ runs in $2^n \cdot \text{poly}(n)$ time — exponential in the seed length. For $P = BPP$, we would need a PRG with seed length $O(\log n)$, producing output of polynomial length $p(n)$. Such a "poly-stretch" PRG with logarithmic seed length would give a polynomial-time deterministic simulation. Constructing such PRGs requires hardness assumptions (e.g., that certain functions require exponential circuits).
