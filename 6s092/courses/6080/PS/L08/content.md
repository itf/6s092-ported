# Polynomial Time

The complexity class **P** (polynomial time) is the class of decision problems solvable by a Turing machine in $O(n^k)$ steps for some constant $k$. P is a robust definition: it is insensitive to the choice of reasonable machine model (single-tape vs multi-tape TM, RAM model, etc.) up to polynomial factors. Problems in P include sorting, shortest paths, primality testing (AKS, 2002), linear programming, and computing GCDs.

The lecture also covers *matrix multiplication* as a case study in algorithmic progress. Naive matrix multiplication of two $n \times n$ matrices takes $O(n^3)$ time. Strassen's 1969 algorithm reduces this to $O(n^{2.81})$ by splitting each matrix into four blocks and using only 7 recursive multiplications instead of 8. The current best algorithms run in $O(n^{2.37...})$ time, and it is conjectured that $O(n^2)$ may be achievable (since reading the input already takes $\Omega(n^2)$ time).

## The Class P

P is defined as $\bigcup_{k \geq 0} \text{DTIME}(n^k)$. The key robustness property: a $k$-tape Turing machine running in time $T(n)$ can be simulated by a single-tape TM in time $O(T(n)^2)$, so polynomial time is preserved. Algorithms in P include: Euclid's GCD in $O(\log^2 n)$ bit operations, sorting in $O(n \log n)$, and Dijkstra's shortest path in $O((V + E) \log V)$.

## Beyond P: The Structure of Efficient Algorithms

Some problems sit at the boundary of P. **Factoring** (given $N$, find a non-trivial factor) is widely believed to be outside P but is not known to be NP-complete either — it seems to be an intermediate problem. AKS primality testing (2002) showed that *testing* primality is in P, even though factoring appears hard. This separation is important: knowing a number is composite does not tell you its factors.

<checkyourself>
Why does the fact that primality is in P not imply that factoring is in P?

<showhide>
Primality testing only asks "is $N$ prime?" — a yes/no question. Factoring asks "find a factor of $N$" — a search problem. The AKS algorithm certifies that $N$ is composite without revealing any factor. Finding a factor seems to require much more work. These are different problems, and membership of one in P says nothing about the other.
