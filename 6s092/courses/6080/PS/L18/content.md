# Cryptographic Protocols

Beyond encryption, cryptography enables remarkable *social* protocols: zero-knowledge proofs, commitment schemes, and secure multi-party computation. A **zero-knowledge proof** convinces a verifier that a statement is true without revealing *why* it is true or any other information beyond the fact of truth. The classic example: proving two graphs are non-isomorphic without revealing anything else.

A **commitment scheme** lets a party commit to a value without revealing it, and later reveal it in a binding way. The online gambling commitment scheme from Lecture 1 is an example. These primitives can be composed to build protocols that allow multiple parties to jointly compute a function of their private inputs without any party learning more than the output — *secure multi-party computation*.

## Zero-Knowledge Proofs

In a zero-knowledge proof for graph non-isomorphism: the verifier sends a random permutation of one of the two graphs; the prover (who knows whether they're isomorphic) identifies which graph was sent. If the graphs are non-isomorphic, the prover can always answer correctly; if they are isomorphic, the prover can do no better than guessing (50% error per round). After $k$ rounds, a cheating prover succeeds with probability $2^{-k}$.

## Interactive Proofs and IP = PSPACE

The formal study of zero-knowledge protocols led to the complexity class **IP** (Interactive Proofs): problems for which a computationally unbounded prover can convince a probabilistic polynomial-time verifier, with completeness (prover succeeds when telling the truth) and soundness (cheating prover fails with probability $\geq 1/2$). Shamir (1990) proved the stunning result **IP = PSPACE**: interactive proofs can verify any PSPACE statement, far beyond NP.

<checkyourself>
What is the difference between an NP proof and an interactive proof (IP)? What can IP verify that NP cannot (assuming NP ≠ PSPACE)?

<showhide>
An NP proof is a static string (certificate) that the verifier checks in polynomial time. An IP proof is an interactive dialogue: the verifier sends random challenges, the prover responds, and the verifier decides based on the entire transcript. With interaction and randomness, the verifier can be convinced of statements like graph non-isomorphism (believed to be outside NP) and all of PSPACE. The randomness in the verifier's challenges prevents the prover from pre-computing all answers.
