# Complexity

Knowing that a problem is *computable* is often not enough — we also need to know whether it's computable *efficiently*. Dijkstra's colleague's objection ("there are finitely many paths, so enumerate them") is correct in theory but useless in practice: enumeration takes exponential time. Computational complexity theory asks which problems can be solved within *reasonable* resource bounds, typically polynomial time.

**Circuit complexity** offers one approach. Given a Boolean function $f: \{0,1\}^n \to \{0,1\}$, how many gates are required to compute it? Shannon's counting argument shows that most $n$-input Boolean functions require $\Omega(2^n / n)$ gates — but finding an *explicit* function with superpolynomial circuit complexity remains one of the great open problems of TCS (closely related to $P \neq NP$).

## Circuit Complexity

To compute XOR of $n$ bits, $n-1$ two-input XOR gates suffice (chain them together). A lower bound argument: to have the output depend on all $n$ inputs, at least $n-1$ gates are necessary — each gate has at most 2 inputs, so the "information" from each input must travel through at least one gate on a path to the output.

## The Time Hierarchy Theorem

The **Time Hierarchy Theorem** says: for any time-constructible function $f(n) \geq n$, there exist problems solvable in $O(f(n))$ time but not in $o(f(n)/\log f(n))$ time. In particular, DTIME$(n^3) \supsetneq$ DTIME$(n^2)$: more time genuinely buys more computational power. The proof uses a diagonalization: a machine that simulates all time-$f(n)$ machines in turn and does the opposite.

<checkyourself>
Why can't we use the Time Hierarchy Theorem to separate P from NP?

<showhide>
The Time Hierarchy Theorem separates deterministic time classes: it tells us DTIME$(n^2) \subsetneq$ DTIME$(n^3)$, etc. But NP is defined in terms of *nondeterministic* computation. The theorem doesn't directly say anything about whether nondeterminism gives extra power over determinism. We need separate techniques (like circuit lower bounds or algebraic methods) to tackle P vs NP.
