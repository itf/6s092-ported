# Bipartite Matching and Assignment Problem

Bipartite matching has deep connections to linear programming, and the matching polytope has a beautiful total dual integrality property.

## Maximum Bipartite Matching

A matching in a bipartite graph $G = (L \cup R, E)$ is a set of edges with no shared endpoints. The **LP relaxation** of maximum matching is:
$$\max \sum_{e} x_e \quad \text{s.t.} \quad \sum_{e \ni v} x_e \leq 1 \;\forall v, \quad x_e \geq 0$$

## Hungarian Algorithm

The **Hungarian algorithm** (also known as the Kuhn-Munkres algorithm) solves the weighted bipartite matching (assignment) problem in $O(n^3)$ time.

<checkyourself>
Explain the augmenting path characterization of maximum matching: a matching is maximum if and only if there is no augmenting path.

<showhide>
An **augmenting path** is a path from an unmatched vertex in $L$ to an unmatched vertex in $R$, alternating between unmatched and matched edges. If such a path exists, we can flip matched/unmatched edges along it to get a matching one size larger. If no augmenting path exists, we can prove the matching is maximum using König's theorem (max matching = min vertex cover in bipartite graphs), which implies no improvement is possible.
</showhide>
</checkyourself>

What is the matching polytope for bipartite graphs, and why does integrality of LP solutions make the matching problem easy to solve?

<showhide>
The bipartite matching polytope is the convex hull of all (0/1) incidence vectors of matchings. Because the vertex-edge incidence matrix is TU, the LP relaxation (which adds $x_e \geq 0$, $\sum_{e \ni v} x_e \leq 1$) is exactly this polytope. The extreme points of the LP feasible region are all 0/1 vectors (matchings), so LP solvers always return an integer solution. This means we can use LP solvers directly for matching, or apply combinatorial algorithms that implicitly solve the LP.
</showhide>
