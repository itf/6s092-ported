# Locality Sensitive Hashing (LSH)

Locality sensitive hashing provides a randomized approach to approximate nearest neighbor search in high dimensions, where exact methods suffer from the curse of dimensionality.

## LSH Definition and Hamming Distance

A hash family $\mathcal{H}$ is **$(r, cr, p_1, p_2)$-sensitive** if for any $x, y$:
- If $d(x,y) \leq r$: $\Pr_h[h(x)=h(y)] \geq p_1$
- If $d(x,y) \geq cr$: $\Pr_h[h(x)=h(y)] \leq p_2$

For Hamming distance in $\{0,1\}^d$: the family $h_i(x) = x_i$ (pick a random bit) gives collision probability $1 - d_H(x,y)/d$.

## Amplification

To reduce false positive/negative rates, we use:
- **AND-construction**: use $k$ hash functions, declare collision only if all agree. Collision prob: $p^k$.
- **OR-construction**: use $L$ AND-groups, declare collision if any group agrees. Collision prob: $1-(1-p^k)^L$.

<checkyourself>
Given an LSH family with $p_1 = 0.8$ and $p_2 = 0.4$, describe how to choose $k$ (AND) and $L$ (OR) to make the collision probability for near neighbors close to 1 and for far neighbors close to 0.

<showhide>
After AND with $k$ functions: near prob = $0.8^k$, far prob = $0.4^k$. After OR with $L$ groups: near prob = $1-(1-0.8^k)^L$, far prob = $1-(1-0.4^k)^L$.

We want to maximize the ratio. Set $k$ so that $0.8^k$ is not too small and $0.4^k$ is very small. E.g., $k = 10$: near = $0.107$, far = $0.0001$. Then $L = 1000$: near = $1-(1-0.107)^{1000} \approx 1$, far = $1-(1-0.0001)^{1000} \approx 0.095$. Tune $k$ to balance: roughly $k = \log(1/p_1)/\log(1/p_2)$ to equalize sensitivity.
</showhide>
