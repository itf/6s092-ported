# Semidefinite Programming Basics

Semidefinite programming (SDP) generalizes linear programming to optimization over positive semidefinite matrices, enabling powerful relaxations of combinatorial problems.

## SDP Formulation

An SDP in **primal form** is:
$$\min \langle C, X \rangle \quad \text{s.t.} \quad \langle A_i, X \rangle = b_i \; \forall i, \quad X \succeq 0$$
where $\langle C, X \rangle = \text{tr}(C^T X) = \sum_{i,j} C_{ij} X_{ij}$, and $X \succeq 0$ means $X$ is positive semidefinite.

## SDP vs LP

SDPs generalize LPs: by restricting $X$ to be diagonal, we get an LP. SDPs can model inner product constraints between vectors and are the natural relaxation for problems involving vector assignments.

<checkyourself>
How does SDP relate to the concept of "vector programs"? Describe how to write MAX-CUT as a vector program, then as an SDP.

<showhide>
A **vector program** assigns a unit vector $v_i \in \mathbb{R}^n$ to each vertex and maximizes $\frac{1}{2}\sum_{(i,j)\in E}(1 - v_i \cdot v_j)$ (since $1 - v_i \cdot v_j = 0$ if same side, $= 1$ if opposite sides for $\pm 1$ unit vectors, roughly). This is equivalent to an SDP: let $X_{ij} = v_i \cdot v_j$; then $X$ is PSD with $X_{ii} = 1$. The constraint "$v_i$ is a unit vector" becomes $X_{ii} = 1$. The MAX-CUT objective is $\frac{1}{2}\sum_{(i,j)\in E}(1 - X_{ij})$, which is linear in $X$.
