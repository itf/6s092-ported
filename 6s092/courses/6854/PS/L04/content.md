# k-wise Independence and Perfect Hashing

$k$-wise independent hash families allow us to analyze algorithms requiring stronger independence guarantees. Perfect hashing achieves $O(1)$ worst-case lookup with $O(m)$ space.

## k-wise Independent Hash Families

A hash family $\mathcal{H}: U \to [n]$ is **$k$-wise independent** if for any $k$ distinct keys $x_1,\ldots,x_k$, the hash values $h(x_1),\ldots,h(x_k)$ are uniformly and independently distributed over $[n]^k$. The polynomial construction: $h(x) = (a_0 + a_1 x + \cdots + a_{k-1} x^{k-1}) \bmod p \bmod n$ with random coefficients $a_i \in \mathbb{F}_p$ is $k$-wise independent.

Why is 2-wise independence insufficient for some applications, and give an example where 4-wise independence is needed?

<showhide>
2-wise independence only controls pairwise collision probabilities. For variance calculations of sums of products of indicator variables (e.g., analyzing the variance of the number of collisions), we need 4th moments, requiring 4-wise independence. For example, in the analysis of a streaming algorithm counting distinct elements (like HyperLogLog variants), 4-wise independence ensures the estimator has small variance. With only 2-wise independence, the variance analysis can fail because cross terms $E[X_i X_j X_k X_l]$ are not controlled.

## FKS Perfect Hashing

The **Fredman-Komlós-Szemerédi (FKS)** scheme achieves $O(m)$ space and $O(1)$ worst-case query time:
1. Hash $m$ keys into $m$ first-level buckets. Bucket $i$ gets $m_i$ keys.
2. For bucket $i$, use a second-level table of size $m_i^2$ with a collision-free hash.
