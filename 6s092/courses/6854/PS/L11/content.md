# Primal-Dual Algorithms

The primal-dual schema is a powerful technique for designing efficient exact and approximation algorithms by maintaining primal and dual solutions simultaneously.

## Complementary Slackness

For LP $\max c^T x$ s.t. $Ax \leq b, x \geq 0$ with dual $\min b^T y$ s.t. $A^T y \geq c, y \geq 0$, **complementary slackness** says primal $x^*$ and dual $y^*$ are both optimal iff:
- $y_i^*(b_i - (Ax^*)_i) = 0$ for all $i$ (dual: if constraint $i$ not tight, $y_i^* = 0$)
- $x_j^*((A^T y^*)_j - c_j) = 0$ for all $j$ (primal: if $x_j^* > 0$, dual constraint $j$ must be tight)

<showhide>

The dual assigns a "distance" $d_v$ to each vertex, with dual constraints $d_v - d_u \leq w(u,v)$ for each edge. Dijkstra's algorithm maintains $d_v$ (shortest path estimates) and grows the set $S$ of settled vertices. At each step, it finds the minimum-cost augmenting path (via relaxation), exactly following the primal-dual schema: maintain a dual feasible solution and make primal feasible by augmenting.
