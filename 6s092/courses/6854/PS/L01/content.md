# Hashing and Load Balancing

Hashing maps keys from a large universe to a small range. A good hash family ensures that even an adversary cannot predict collisions.

## 2-Universal Hash Families

A family $\mathcal{H}$ is **2-universal** if for any two distinct keys $x \neq y$, $\Pr_{h \in \mathcal{H}}[h(x) = h(y)] \leq 1/n$. The standard construction is $h_{a,b}(x) = (ax + b \bmod p) \bmod n$ where $p$ is a prime larger than the universe size.

<checkyourself>
Suppose you have $m$ items hashed into $n$ buckets using a 2-universal hash family. Derive the expected number of colliding pairs (pairs $(i,j)$ with $i \neq j$ that hash to the same bucket).

<showhide>
There are $\binom{m}{2} = m(m-1)/2$ pairs. Each pair collides with probability at most $1/n$ (by 2-universality). By linearity of expectation, the expected number of collisions is at most $m(m-1)/(2n)$.
</showhide>
</checkyourself>

<checkyourself>
Why is it important to use a hash family (random choice of hash function) rather than a single fixed hash function when dealing with adversarial inputs?

<showhide>
A fixed hash function can be exploited: an adversary who knows the function can choose all $m$ inputs to map to the same bucket, causing $O(m)$ lookup time. With a random hash from a 2-universal family, no adversary can force more than $O(1)$ expected collisions per bucket, since the adversary must choose inputs before seeing the hash function.
</showhide>
