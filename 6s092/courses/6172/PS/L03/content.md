# Bit Hacks

Modern processors execute bitwise operations in a single cycle. Exploiting bit-level representation allows many computations to be done much faster than with ordinary arithmetic. A $w$-bit word stores an unsigned integer with value $\sum_{i=0}^{w-1} x_i \cdot 2^i$, or a signed (two's complement) integer where the most significant bit has weight $-2^{w-1}$.

Two's complement is the standard signed-integer representation on virtually all modern hardware. The key property is that the same hardware addition/subtraction circuits work for both signed and unsigned arithmetic. The negative of $x$ in two's complement is $\sim x + 1$ (bitwise NOT plus 1).

## Basic Bit Operations

Common bit tricks and their uses:

| Operation | Expression | Notes |
|---|---|---|
| Set bit $k$ | `x \|= (1 << k)` | |
| Clear bit $k$ | `x &= ~(1 << k)` | |
| Test bit $k$ | `(x >> k) & 1` | |
| Round up to power of 2 | see below | |
| Isolate lowest set bit | `x & (-x)` | `-x` = `~x + 1` |
| Clear lowest set bit | `x & (x-1)` | also counts set bits |

The expression `x & (x-1)` clears the lowest set bit of `x`. This is useful for counting set bits (popcount) via Brian Kernighan's algorithm:

```c
int popcount(unsigned int x) {
    int count = 0;
    while (x) { x &= x - 1; count++; }
    return count;
}
```

<checkyourself>
What does `x & (-x)` compute, and why? (Assume two's complement representation.)

<showhide>
`x & (-x)` isolates the lowest set bit of `x`. In two's complement, `-x = ~x + 1`. The addition of 1 to `~x` causes a carry that propagates through all the trailing zeros of `~x` (which correspond to the trailing zeros of `x`), setting the bit at the position of `x`'s lowest set bit. The AND with `x` then keeps only that bit.

## No-Branch Swap and Absolute Value

A classic bit hack avoids a branch for computing absolute value:

```c
// Branchless absolute value (32-bit)
int abs_val(int x) {
    int mask = x >> 31;       // arithmetic shift: all 0s or all 1s
    return (x + mask) ^ mask; // if x >= 0: (x+0)^0 = x
                               // if x < 0: (x+(-1))^(-1) = (-x-1)^(-1) = -x
}
```

Similarly, a no-branch swap using XOR:

```c
void swap(int *a, int *b) {
    *a ^= *b;
    *b ^= *a;
    *a ^= *b;
}
```

Note: this XOR swap is **not** faster than a temporary-variable swap on modern CPUs due to data dependencies, but it is a cute bit hack.

How can you round an integer `n` up to the nearest power of two using bit operations?

<showhide>
For a 32-bit unsigned integer:
```c
uint32_t next_pow2(uint32_t n) {
    n--;
    n |= n >> 1; n |= n >> 2; n |= n >> 4;
    n |= n >> 8; n |= n >> 16;
    return n + 1;
}
```
The idea: decrement first (to handle the case where `n` is already a power of 2), then propagate all set bits downward so the lower bits all become 1, then add 1 to get the next power of 2.
</showhide>
