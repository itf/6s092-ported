# Assembly Language and Computer Architecture

Understanding assembly language helps performance engineers see what the compiler actually produces, identify missed optimizations, and reason about low-level behavior such as instruction latency and throughput. The x86-64 ISA is the dominant architecture for desktop and server computing; fluency in reading (not necessarily writing) x86-64 assembly is a valuable skill.

The compilation pipeline takes C source → preprocessed source → LLVM IR → assembly → machine code. At each stage, information is transformed. Assembly is the human-readable representation of machine code, and understanding it reveals exactly what operations the CPU will execute.

## x86-64 Basics

x86-64 has 16 general-purpose 64-bit registers: `rax`, `rbx`, `rcx`, `rdx`, `rsi`, `rdi`, `rsp`, `rbp`, `r8`–`r15`. Sub-registers (`eax` = low 32 bits of `rax`, `ax` = low 16 bits, `al` = low 8 bits) can be accessed for narrower operations.

Key instruction classes:
- **Data movement**: `mov`, `movzx` (zero-extend), `movsx` (sign-extend), `lea`
- **Arithmetic/logic**: `add`, `sub`, `imul`, `and`, `or`, `xor`, `shl`, `shr`, `sar`
- **Control flow**: `jmp`, `je`/`jz`, `jne`, `jl`, `jge`, conditional moves (`cmov`)
- **Stack**: `push`, `pop`, `call`, `ret`

The `lea` (load effective address) instruction is often used for arithmetic rather than memory access:
```asm
lea rax, [rdi + rdi*2]   ; rax = rdi + rdi*2 = 3*rdi (no memory access)
```

What is the difference between `mov rax, [rbx]` and `mov rax, rbx` in x86-64 assembly?

<showhide>
`mov rax, rbx` copies the 64-bit value in register `rbx` directly into `rax` — a register-to-register move. `mov rax, [rbx]` treats the value in `rbx` as a memory address and loads the 64-bit value from that address into `rax` — a memory load. The square brackets indicate a memory dereference (like `*ptr` in C).

## Instruction Latency and Throughput

Modern x86-64 CPUs are superscalar and out-of-order. Each instruction has a **latency** (how many cycles before its result is available) and a **throughput** (how many can start per cycle on the execution units). For example, a 64-bit integer multiply (`imul`) on recent Intel CPUs has latency 3 cycles and throughput 1 per cycle.

The difference between latency-bound and throughput-bound code matters greatly. If operations are chained (each depends on the previous result), the bottleneck is latency. If operations are independent, the bottleneck is throughput.

What is the x86-64 calling convention for the first six integer/pointer function arguments?

<showhide>
In the System V AMD64 ABI (used on Linux/macOS), the first six integer or pointer arguments are passed in registers: `rdi`, `rsi`, `rdx`, `rcx`, `r8`, `r9`, in that order. Additional arguments are passed on the stack. The return value is in `rax` (and `rdx` for 128-bit returns). Caller-saved registers include `rax`, `rcx`, `rdx`, `rsi`, `rdi`, `r8`–`r11`; callee-saved registers include `rbx`, `rbp`, `r12`–`r15`.
</showhide>
