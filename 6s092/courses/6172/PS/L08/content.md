# Analysis of Multithreaded Algorithms

The performance of a parallel program is characterized by two quantities: **work** $T_1$ (the total time on one processor, equal to the serial execution time) and **span** $T_\infty$ (the critical-path length, the minimum time on infinitely many processors). These determine the performance on $P$ processors via:

$$T_P \geq \max\!\left(\frac{T_1}{P},\, T_\infty\right)$$

The work law says $T_P \geq T_1/P$ (can't be faster than dividing work evenly), and the span law says $T_P \geq T_\infty$ (can't be faster than the critical path). The work-stealing scheduler achieves $T_P \leq T_1/P + O(T_\infty)$ in expectation.

## Work and Span of Parallel Algorithms

The **speedup** on $P$ processors is $T_1 / T_P$. The **parallelism** (maximum useful speedup) is $T_1 / T_\infty$.

For the parallel Fibonacci example:
- Work: $T_1(n) = T(n) = \Theta(\phi^n)$ where $\phi = (1+\sqrt{5})/2$ (same as serial).
- Span: $T_\infty(n) = T_\infty(n-1) + \Theta(1)$ (only the longer branch), giving $T_\infty(n) = \Theta(n)$.
- Parallelism: $T_1/T_\infty = \Theta(\phi^n / n)$ — exponentially large for large $n$.

<checkyourself>
Why is the span of parallel Fibonacci $\Theta(n)$ rather than $\Theta(\log n)$ or $\Theta(\phi^n)$?

<showhide>
The span is the length of the critical path—the longest chain of dependent operations. In `cilk_spawn fib(n-1)`, the spawned call `fib(n-1)` and the continuation `fib(n-2)` run in parallel. The critical path goes through the *longer* call, which is `fib(n-1)`. The recurrence is $T_\infty(n) = T_\infty(n-1) + O(1)$, giving $T_\infty(n) = O(n)$. It's $\Theta(n)$, not $\Theta(\phi^n)$ because we always follow the longer branch.
</showhide>

## Divide-and-Conquer Recurrences

For parallel divide-and-conquer algorithms, work and span satisfy separate recurrences. For parallel merge sort splitting into 2 subproblems of size $n/2$ with $O(n)$ merge:

- **Work**: $T_1(n) = 2T_1(n/2) + \Theta(n) = \Theta(n \log n)$ (same as serial)
- **Span**: If merge is parallelized to $O(\log^2 n)$: $T_\infty(n) = T_\infty(n/2) + \Theta(\log^2 n) = \Theta(\log^3 n)$
- **Parallelism**: $\Theta(n \log n / \log^3 n) = \Theta(n / \log^2 n)$

The **Master Theorem** applies: for $T(n) = aT(n/b) + f(n)$, compare $f(n)$ to $n^{\log_b a}$:
1. If $f(n) = O(n^{\log_b a - \epsilon})$: $T(n) = \Theta(n^{\log_b a})$
2. If $f(n) = \Theta(n^{\log_b a})$: $T(n) = \Theta(n^{\log_b a} \log n)$
3. If $f(n) = \Omega(n^{\log_b a + \epsilon})$ and regularity holds: $T(n) = \Theta(f(n))$

<checkyourself>
State the work law and span law for parallel programs. What do they tell you about $T_P$?

<showhide>
**Work law**: $T_P \geq T_1 / P$. Even if $P$ processors each do equal work with no overhead, the execution time can't be less than total work divided by $P$.
**Span law**: $T_P \geq T_\infty$. The critical path (longest chain of serial dependencies) sets a lower bound on execution time, regardless of how many processors are available.
Together they give: $T_P \geq \max(T_1/P, T_\infty)$. The work-stealing scheduler nearly achieves this bound.
</showhide>
