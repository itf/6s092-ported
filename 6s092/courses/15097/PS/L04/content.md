# Bias/Variance Tradeoff and Inference

Statistical inference involves estimating a parameter $\theta$ of a distribution from data. The quality of an estimator $\hat{\theta}$ is characterized by its **bias** and **variance**. Understanding this tradeoff is fundamental to selecting good models in machine learning.

## Bias and Variance of an Estimator

Given estimator $\hat{\theta}$ for true parameter $\theta$:
- **Bias:** $\text{Bias}(\hat{\theta}, \theta) = E[\hat{\theta}] - \theta$
- **Variance:** $\text{Var}(\hat{\theta}) = E[(\hat{\theta} - E[\hat{\theta}])^2]$
- **Mean Squared Error:** $\text{MSE}(\hat{\theta}) = \text{Bias}^2 + \text{Var}(\hat{\theta})$

An estimator is **unbiased** if $E[\hat{\theta}] = \theta$.

## The Bias-Variance Tradeoff in Model Selection

As model complexity increases: bias tends to decrease (the model can represent more complex patterns) and variance tends to increase (more parameters to estimate from limited data). The optimal complexity minimizes total MSE = Bias² + Variance.

## Confidence Intervals

A **confidence interval** provides a range of plausible values for an unknown parameter. A 95% CI means: if we repeated the experiment many times, 95% of the constructed intervals would contain the true parameter. For a normal distribution with known variance, the 95% CI for the mean is $\bar{X} \pm 1.96 \cdot \frac{\sigma}{\sqrt{m}}$.

<checkyourself>
What is the difference between a confidence interval and a Bayesian credible interval?

<showhide>
A **confidence interval** (frequentist) means: if we repeated the data collection and CI construction procedure many times, 95% of the intervals would contain the true (fixed, unknown) parameter. The parameter itself is not random in this view. A **credible interval** (Bayesian) treats the parameter as a random variable with a prior distribution, and the 95% credible interval means: given the data, there is 95% posterior probability that the parameter lies in the interval. These are philosophically different but often numerically similar.
</showhide>
</checkyourself>
