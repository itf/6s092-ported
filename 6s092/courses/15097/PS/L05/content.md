# Clustering (K-Means)

**Clustering** is an unsupervised method that partitions data into groups (clusters) such that points within a cluster are more similar to each other than to points in other clusters. The most popular algorithm is **K-Means**, which minimizes the total within-cluster squared distance to cluster centers.

## K-Means Algorithm

**Input:** points $\{x_i\}_{i=1}^m \subset \mathbb{R}^n$, number of clusters $K$.

**Goal:** Find centers $z_1, \ldots, z_K$ minimizing:
$$\text{cost}(z_1, \ldots, z_K) = \sum_{i=1}^m \min_k \|x_i - z_k\|_2^2$$

**Algorithm (Lloyd's):**
1. Initialize $K$ cluster centers (e.g., randomly).
2. **Assignment step:** Assign each $x_i$ to its nearest center $z_k$.
3. **Update step:** Set each $z_k$ to the mean of points assigned to cluster $k$.
4. Repeat until convergence.

<checkyourself>
K-Means is guaranteed to converge (the cost decreases or stays the same at each step), but it is NOT guaranteed to find the global minimum. Why?

<showhide>
K-Means performs alternating minimization: the assignment step minimizes cost over assignments (given fixed centers), and the update step minimizes cost over centers (given fixed assignments). Each step decreases (or maintains) the cost. Since there are finitely many assignments, the algorithm must terminate. However, it converges to a **local** minimum that depends on the initialization — different starting points can yield different (suboptimal) clusterings. Running K-Means multiple times with different random initializations and keeping the best result is a common remedy.
</showhide>
</checkyourself>

## Voronoi Partitions

Given centers $z_1, \ldots, z_K$, the **Voronoi partition** assigns each point in space to the nearest center. The clusters $C_k = \{x_i : \text{nearest center is } z_k\}$ form the Voronoi cells. The K-Means assignment step computes this partition.

## Choosing K

K-Means requires specifying $K$ in advance. Common heuristics for choosing $K$:
- **Elbow method:** Plot cost vs. $K$; look for an "elbow" where cost drops sharply before flattening.
- **Silhouette score:** Measures how well each point fits its cluster relative to neighboring clusters.
- **Domain knowledge:** Prior knowledge about the number of meaningful groups.

<checkyourself>
Why does K-Means cost always decrease as $K$ increases? Is maximizing $K$ a good strategy?

<showhide>
Adding more clusters can never increase the optimal K-Means cost, because we can always reproduce the $K$-cluster solution using $K+1$ clusters (by splitting any cluster into two with the same total cost, or setting one center equal to an existing one). At the extreme $K = m$, each point is its own cluster and cost is zero. But maximum $K$ is useless — the goal of clustering is to find a compact, interpretable summary. We want the smallest $K$ that captures meaningful structure.
</showhide>
</checkyourself>
