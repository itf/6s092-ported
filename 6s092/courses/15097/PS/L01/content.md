# Rule Mining and the Apriori Algorithm

Rule mining discovers associations in large transaction datasets. The canonical example is market basket analysis: given a database of purchases, find rules like "customers who buy diapers also buy beer." The Apriori algorithm is the classic approach for efficiently finding all frequent itemsets and strong association rules.

## Key Definitions

An **itemset** is a subset of items. The **support** of an itemset $A$ is the number (or fraction) of transactions containing $A$. The **confidence** of a rule $A \rightarrow B$ is $\text{Conf}(A \rightarrow B) = \frac{\text{Supp}(A \cup B)}{\text{Supp}(A)}$, which estimates $\hat{P}(B|A)$.

A rule is **strong** if $\text{Supp}(A \cup B) \geq \theta$ (minimum support) and $\text{Conf}(A \rightarrow B) \geq \text{minconf}$.

<checkyourself>
Why does the Apriori algorithm use a "breadth-first search" strategy, generating $k$-itemsets from $(k-1)$-itemsets rather than checking all subsets directly?

<showhide>
Because of the **downward closure (anti-monotonicity) property**: if an itemset is frequent, all of its subsets must also be frequent. Equivalently, if any subset of an itemset is infrequent, the whole itemset is infrequent and can be pruned without counting. By building candidate $k$-itemsets only from frequent $(k-1)$-itemsets, Apriori avoids examining exponentially many infrequent supersets.
</showhide>
</checkyourself>

## Downward Closure Property

The support of a set is always at most the support of any of its subsets: $\text{Supp}(A \cup B) \leq \text{Supp}(A)$. This means frequent itemsets form a downward-closed family: if $S$ is frequent, every $S' \subseteq S$ is also frequent.

## Generating Rules from Frequent Itemsets

Once all frequent itemsets are found, strong rules are generated by examining all non-trivial splits of each frequent itemset $\ell$ into antecedent $a$ and consequent $\ell \setminus a$, keeping those with $\text{Supp}(\ell) / \text{Supp}(a) \geq \text{minconf}$.

<checkyourself>
For a frequent itemset $\ell$ of size $k$, how many candidate rules $a \rightarrow (\ell \setminus a)$ must be checked (where $a$ is non-empty and $a \neq \ell$)?

<showhide>
We need all non-empty proper subsets $a \subsetneq \ell$. There are $2^k - 2$ such subsets (all subsets minus the empty set and $\ell$ itself). For example, for $\ell = \{A, B, C\}$ (k=3), there are $2^3 - 2 = 6$ candidate rules.
</showhide>
</checkyourself>
