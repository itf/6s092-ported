# Cache-Oblivious Algorithms: Medians & Matrices

The **cache-oblivious model** (Frigo, Leiserson, Prokop, Ramachandran 1999) analyzes algorithms without knowing cache parameters (cache size $M$ and block/line size $B$). An algorithm is **cache-oblivious** if it achieves optimal cache complexity without tuning to specific $M$ and $B$ values. This is powerful because such an algorithm is simultaneously optimal for all levels of the memory hierarchy.

The model counts **cache misses** (also called "block transfers" or "I/Os"): the number of blocks moved between slow memory and cache. For an array of $N$ elements, a sequential scan incurs $\Theta(N/B)$ cache misses (optimal). For comparison, random access can incur $\Theta(N)$ misses (one per element).

**Cache-oblivious matrix multiplication**: the naive $O(N^3)$ algorithm has poor cache performance. A divide-and-conquer approach (recurse on $N/2 \times N/2$ submatrices) achieves $\Theta(N^3/(B\sqrt{M}))$ cache misses—matching the optimal cache-aware algorithm, without knowing $M$ or $B$.

## Cache Complexity Analysis

The cache complexity of an algorithm is measured in the **ideal cache model**: two-level memory (cache of size $M$, main memory infinite), fully associative cache with optimal replacement (offline LRU). The number of cache misses for a problem of size $N$ is denoted $Q(N; M, B)$.

For **matrix multiplication** ($N \times N$ matrices), the naive algorithm multiplies rows of $A$ by columns of $B$: each column access causes a cache miss unless the column fits in cache. The divide-and-conquer algorithm recurses until submatrices fit in cache ($N^2 \leq M/3$, i.e., $N \leq \sqrt{M/3}$), then multiplies in-cache. The recurrence: $Q(N) = 8Q(N/2) + O(N^2/B)$, solving to $Q(N) = O(N^3/(B\sqrt{M}))$.

<showhide>
A cache-oblivious algorithm achieves optimal cache performance without being parameterized by $M$ (cache size) or $B$ (block size). It uses the same code for all cache configurations. This is desirable because: (1) actual $M$ and $B$ vary across machines and cache levels; (2) a single algorithm is optimal at every level of a multi-level memory hierarchy simultaneously; (3) no tuning or recompilation is needed. By contrast, cache-aware algorithms (like tiling/blocking) require knowing $M$ and $B$ to choose tile sizes.

## Cache-Oblivious Median Finding

The cache-oblivious **selection** algorithm (finding the $k$-th smallest element) achieves $O(N/B)$ cache misses—optimal, matching sequential scan. The key is to use a cache-oblivious sorting step or a carefully designed recursion that processes contiguous memory.

For **median of medians** (deterministic selection), the cache-oblivious version divides into groups along a space-filling curve or uses a two-level divide strategy to ensure $O(N/B)$ cache misses, compared to $O(N)$ for the naive implementation (which accesses every element individually).

What is the "tall cache assumption" and why is it needed for cache-oblivious analysis?

<showhide>
The **tall cache assumption** requires $M = \Omega(B^2)$ (equivalently, the cache is "tall": it holds at least $B$ full blocks). This is needed for cache-oblivious analysis because it ensures that when a subproblem fits in cache ($N^2 = O(M)$), the corresponding blocks also fit (blocks of size $B$ for a problem of size $\sqrt{M}$ need $\sqrt{M}/B = O(M/(B^2)) \cdot B = O(M/B)$ blocks, which fit in $M/B$ cache lines). Without this assumption, some cache-oblivious bounds don't hold.
</showhide>
