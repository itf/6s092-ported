# Complexity: Approximation Algorithms

For NP-hard optimization problems, we seek **approximation algorithms**: efficient algorithms that produce solutions within a provable factor of the optimal. An algorithm is an **$\alpha$-approximation** if for all inputs, $\text{ALG} \leq \alpha \cdot \text{OPT}$ (for minimization; $\geq 1/\alpha \cdot \text{OPT}$ for maximization), where $\alpha \geq 1$.

Classic examples:
- **Vertex Cover**: 2-approximation via maximal matching (take both endpoints of each matching edge).
- **Set Cover**: $O(\log n)$-approximation via greedy (always pick the set covering the most uncovered elements); this is optimal unless P = NP.
- **TSP** (metric): 2-approximation via MST-based algorithm; **Christofides' algorithm** achieves $3/2$-approximation.
- **Knapsack**: FPTAS (fully polynomial-time approximation scheme) achieving $(1-\epsilon)$-approximation in $O(n^2/\epsilon)$ time.

## Vertex Cover: 2-Approximation

**Algorithm**: find a maximal matching $M$ (greedily add edges that don't share endpoints); return all endpoints of $M$ as the vertex cover.

**Analysis**: Let $C^*$ be the optimal vertex cover. Each edge in $M$ must be covered by $C^*$, and all edges in $M$ are vertex-disjoint, so $|C^*| \geq |M|$. The algorithm returns $2|M|$ vertices. Thus $|C| = 2|M| \leq 2|C^*|$.

Why can't we improve the vertex cover 2-approximation to a $(2-\epsilon)$-approximation (for any fixed $\epsilon > 0$) assuming the Unique Games Conjecture?

<showhide>
The **Unique Games Conjecture (UGC)** of Khot (2002) implies that vertex cover is hard to approximate within any factor better than 2. Under UGC, for any $\epsilon > 0$, it is NP-hard to achieve a $(2-\epsilon)$-approximation for vertex cover. This matches the trivial 2-approximation from maximal matching, making the analysis tight. Whether UGC is true is itself an open problem.
</showhide>

## TSP Approximation

The **metric TSP** problem: given cities with distances satisfying the triangle inequality, find the minimum-cost Hamiltonian cycle.

**2-approximation**: (1) find MST $T$; (2) double all edges to get an Eulerian multigraph; (3) find an Euler tour; (4) shortcut repeated vertices. Cost $\leq 2 \cdot \text{MST} \leq 2 \cdot \text{OPT}$.

**Christofides' $3/2$-approximation**: (1) find MST $T$; (2) find minimum-weight perfect matching $M$ on odd-degree vertices of $T$; (3) form Eulerian multigraph $T \cup M$; (4) find Euler tour and shortcut. Cost $\leq \text{MST} + |M| \leq \text{OPT} + \text{OPT}/2 = 3/2 \cdot \text{OPT}$.

What is a PTAS (polynomial-time approximation scheme) and how does it differ from an FPTAS?

<showhide>
A **PTAS** gives a $(1+\epsilon)$-approximation for any fixed $\epsilon > 0$ in time polynomial in $n$ (but possibly exponential in $1/\epsilon$, e.g., $O(n^{1/\epsilon})$). An **FPTAS** (fully polynomial) is a PTAS that is also polynomial in $1/\epsilon$, e.g., $O(n^2/\epsilon)$. An FPTAS is strictly stronger: you can set $\epsilon = 1/n$ and still get polynomial time. Knapsack has an FPTAS; Euclidean TSP has a PTAS (Arora's algorithm) but no known FPTAS.
</showhide>
</checkyourself>
