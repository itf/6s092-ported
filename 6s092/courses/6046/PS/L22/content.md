# Cryptography: Encryption

**Encryption** provides confidentiality: a sender encrypts a message $m$ using a key so that only the intended recipient can decrypt it. Modern cryptography is built on **computational hardness assumptions**: security is based on the conjectured difficulty of problems like integer factorization (RSA), discrete logarithm (Diffie-Hellman), and lattice problems.

**Symmetric-key encryption** (e.g., AES) uses the same key for encryption and decryption; it is efficient but requires a shared secret. **Asymmetric (public-key) encryption** (e.g., RSA, ElGamal) uses a public key for encryption and a private key for decryption; it solves the key distribution problem.

**RSA**: choose large primes $p, q$; let $N = pq$; pick $e$ with $\gcd(e, \phi(N)) = 1$ where $\phi(N) = (p-1)(q-1)$; the private key is $d = e^{-1} \bmod \phi(N)$. Encryption: $c = m^e \bmod N$. Decryption: $m = c^d \bmod N$. Security relies on the hardness of factoring $N$ (or the RSA problem).

## RSA and Number Theory

RSA correctness follows from **Euler's theorem**: $m^{\phi(N)} \equiv 1 \pmod{N}$ for $\gcd(m, N) = 1$. Since $ed \equiv 1 \pmod{\phi(N)}$, we have $c^d = (m^e)^d = m^{ed} = m^{1 + k\phi(N)} = m \cdot (m^{\phi(N)})^k \equiv m \pmod{N}$.

**RSA key generation**: generating $p, q$ requires finding large primes, done by Miller-Rabin primality testing (randomized, $O(k \log^2 N)$ per test with error $\leq 4^{-k}$). Computing $d = e^{-1} \bmod \phi(N)$ uses the extended Euclidean algorithm in $O(\log^2 N)$ time.

Why is "textbook RSA" ($c = m^e \bmod N$ directly) insecure, and what padding scheme fixes it?

<showhide>
Textbook RSA is **deterministic**: encrypting the same $m$ always gives the same $c$. An attacker can (1) detect whether two ciphertexts encrypt the same message, and (2) for small message spaces, enumerate all possible ciphertexts. It is also **malleable**: multiplying $c$ by $r^e \bmod N$ gives an encryption of $rm \bmod N$. **OAEP (Optimal Asymmetric Encryption Padding)** fixes this by adding randomness before encryption, achieving CCA2 security in the random oracle model.
</showhide>

## Diffie-Hellman Key Exchange

**Diffie-Hellman (DH) key exchange** allows two parties to establish a shared secret over a public channel:
1. Agree on public prime $p$ and generator $g$ of $\mathbb{Z}_p^*$.
2. Alice picks random $a$, sends $A = g^a \bmod p$.
3. Bob picks random $b$, sends $B = g^b \bmod p$.
4. Shared secret: Alice computes $B^a = g^{ab} \bmod p$; Bob computes $A^b = g^{ab} \bmod p$.

Security relies on the **Computational Diffie-Hellman (CDH) problem**: given $g^a$ and $g^b$, compute $g^{ab}$. The **discrete logarithm problem** (DLP): given $g^a$, find $a$. CDH $\leq$ DLP but the converse is not known.

What is semantic security (IND-CPA) and why is it the standard security definition for encryption?

<showhide>
**IND-CPA** (indistinguishability under chosen-plaintext attack): an encryption scheme is IND-CPA secure if no polynomial-time adversary can distinguish encryptions of two chosen messages $m_0, m_1$ with probability significantly better than $1/2$, even after seeing polynomially many encryptions of its choice. This captures the intuition that ciphertexts reveal no information about plaintexts. Deterministic encryption (like textbook RSA) cannot be IND-CPA secure. Stronger notions: IND-CCA1 (non-adaptive chosen-ciphertext) and IND-CCA2 (adaptive chosen-ciphertext, the standard for practical systems).
</showhide>
