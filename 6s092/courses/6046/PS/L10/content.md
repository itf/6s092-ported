# Dynamic Programming: Advanced DP

**Dynamic programming (DP)** solves optimization problems by combining solutions to overlapping subproblems. This lecture covers advanced DP techniques beyond basic shortest paths and sequence alignment: **DP on intervals**, **DP on trees**, **DP with convex hull trick**, and **bitonic TSP**.

The **optimal BST problem** exemplifies DP on intervals: given keys $k_1 < k_2 < \cdots < k_n$ with search probabilities $p_i$, find a BST minimizing expected search cost. The DP is $e[i,j] = \min_{r=i}^{j} \{e[i,r-1] + e[r+1,j] + w(i,j)\}$, where $w(i,j) = \sum_{k=i}^{j} p_k$. This is $O(n^3)$ naively but can be improved to $O(n^2)$ using Knuth's monotone SMAWK optimization (the optimal root is monotone in the interval endpoints).

The **convex hull trick** (or "CHT") optimizes DP recurrences of the form $\text{dp}[i] = \min_j \{f(j) + g(j) \cdot h(i)\}$ to $O(n \log n)$ or $O(n)$ (from $O(n^2)$) when the lines $y = g(j) \cdot x + f(j)$ can be maintained in a convex hull structure.

## DP on Intervals

Many problems decompose naturally on intervals $[i,j]$. The subproblem DAG for interval DP has $O(n^2)$ nodes; transitions consider all "split points" in the interval, giving $O(n^3)$ total time. Standard examples: matrix chain multiplication, optimal BST, polygon triangulation.

**Matrix chain multiplication**: given $n$ matrices $A_1, A_2, \ldots, A_n$ of dimensions $p_0 \times p_1, p_1 \times p_2, \ldots, p_{n-1} \times p_n$, find the parenthesization minimizing total scalar multiplications. DP: $m[i,j] = \min_{k=i}^{j-1} \{m[i,k] + m[k+1,j] + p_{i-1} p_k p_j\}$, in $O(n^3)$ time.

<checkyourself>
In the matrix chain multiplication DP, why does trying all split points $k$ lead to the correct optimal substructure?

<showhide>
The optimal parenthesization of $A_i \cdots A_j$ must split somewhere: the final multiplication is $(A_i \cdots A_k)(A_{k+1} \cdots A_j)$ for some $k$. The cost of the final multiplication plus the optimal costs of the two sub-chains is the total cost. Since the optimal solution to the whole problem contains optimal solutions to the sub-chains (the optimal substructure property), we can try all $k$ and take the minimum.
</showhide>

## Convex Hull Trick

The **convex hull trick** applies when the DP recurrence has the form $\text{dp}[i] = \min_{j < i} \{a_j \cdot b_i + c_j\}$ (i.e., for each $i$, minimize a linear function in $b_i$ over choices of $j$). Each $j$ defines a line $y = a_j x + c_j$; the minimum over $j$ at $x = b_i$ is the lower envelope of these lines. Maintaining the lower convex hull allows each query/insert in $O(\log n)$ time (or $O(1)$ amortized if $b_i$ is monotone).

Example: divide-and-conquer optimization for the "hiring and firing" DP or the "line container" in competitive programming.

<checkyourself>
What is the "optimal substructure" property, and why is it necessary (but not sufficient) for DP to work?

<showhide>
Optimal substructure means that an optimal solution to a problem contains optimal solutions to its subproblems. It is necessary for DP because we assume we can combine optimal subsolutions to get an optimal overall solution. It is not sufficient on its own—we also need overlapping subproblems (so that DP saves work over plain recursion). Without overlapping subproblems, divide and conquer (without memoization) already works. A counterexample where optimal substructure fails is shortest paths with cycles under certain edge weight conditions.
</showhide>
</checkyourself>
