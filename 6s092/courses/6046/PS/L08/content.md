# Randomization: Universal & Perfect Hashing

**Universal hashing** and **perfect hashing** provide theoretical foundations for hash tables with strong worst-case guarantees. Standard hash tables can degrade to $O(n)$ per operation if many keys hash to the same slot, but with a randomly chosen hash function from a **universal family**, the expected number of collisions is small.

A family $\mathcal{H}$ of hash functions $h: U \to [m]$ is **universal** if for any two distinct keys $x, y \in U$, $\Pr_{h \in \mathcal{H}}[h(x) = h(y)] \leq 1/m$. Under universal hashing, the expected number of keys that collide with any given key $x$ is at most $n/m$ (for $n$ keys in a table of size $m$). With $m = n$, this is $O(1)$ expected collisions per key, giving $O(1)$ expected search time.

**Perfect hashing** achieves $O(1)$ worst-case search time for a static set of $n$ keys. The two-level scheme: hash $n$ keys into $n$ buckets (using a universal hash function); for each bucket $i$ with $n_i$ keys, create a secondary hash table of size $n_i^2$ using another universal function. By the birthday paradox, a random universal function on $n_i$ keys into $n_i^2$ slots has no collisions with probability $\geq 1/2$, so $O(1)$ tries suffice.

## Universal Hash Families

A simple universal family for $U = \{0, 1, \ldots, p-1\}$ (prime $p$) and table size $m$: $h_{a,b}(x) = ((ax + b) \bmod p) \bmod m$ for $a \in \{1, \ldots, p-1\}$, $b \in \{0, \ldots, p-1\}$. For any $x \neq y$, the collision probability is $\leq \lceil p/m \rceil / (p-1) \leq 1/(m-1) + 1/p \approx 1/m$.

The key property: for a universal family, expected lookup time in a chaining hash table with $n$ keys and $m = n$ slots is $O(1)$.

## Perfect Hashing

**Static perfect hashing** (FKS scheme): given $n$ keys, find a collision-free hash function for $O(1)$ worst-case lookup. The two-level approach:
1. Hash all $n$ keys into $n$ primary buckets; expected $O(n)$ total slots.
2. For bucket $i$ with $n_i$ keys, allocate $n_i^2$ secondary slots; by the birthday bound, a random universal function has no collisions with probability $\geq 1/2$.

The total secondary space: $\sum_i n_i^2 = O(n)$ in expectation (because $\mathbb{E}[\sum n_i^2] \leq 2n$ under a universal primary hash). Construction time is $O(n)$ expected.

What is the difference between a Las Vegas and Monte Carlo randomized algorithm in the context of hash table construction?

<showhide>
A Las Vegas algorithm always produces a correct result but has a random running time (e.g., we retry until we find a collision-free hash—expected $O(1)$ retries). A Monte Carlo algorithm has a fixed runtime but may produce an incorrect result with some (small) probability. Perfect hashing construction is Las Vegas: we repeatedly sample hash functions until one with no collisions is found, which happens in $O(1)$ expected trials.
