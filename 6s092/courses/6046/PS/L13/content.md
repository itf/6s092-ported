# Incremental Improvement: Max Flow, Min Cut

**Network flow** is one of the most versatile tools in algorithm design. The **maximum flow problem**: given a directed graph $G = (V, E)$ with capacities $c(u,v) \geq 0$, source $s$, and sink $t$, find a flow $f: E \to \mathbb{R}_{\geq 0}$ satisfying capacity and conservation constraints, maximizing $|f| = \sum_{v} f(s,v)$.

The **Ford-Fulkerson method** incrementally improves the flow by finding **augmenting paths** in the **residual graph** $G_f$. The residual graph has forward edges with remaining capacity and backward edges with current flow. The algorithm terminates when no augmenting path exists, at which point the current flow is maximum by the **Max-Flow Min-Cut Theorem**.

**Max-Flow Min-Cut Theorem**: The maximum flow from $s$ to $t$ equals the minimum capacity of any **$s$-$t$ cut** $(S, T)$ where $s \in S$, $t \in T$, and cut capacity $= \sum_{u \in S, v \in T} c(u,v)$. This is a fundamental duality result.

## Ford-Fulkerson and Edmonds-Karp

**Ford-Fulkerson**: find any augmenting path in $G_f$, push flow equal to the bottleneck capacity along it, update residual graph. With integer capacities, each augmentation increases $|f|$ by at least 1, so it terminates in $O(|f^*|)$ augmentations, giving $O(|f^*| \cdot m)$ total time—which can be exponential in edge weights.

**Edmonds-Karp**: use BFS to find the **shortest** augmenting path (fewest edges). This gives $O(nm)$ augmentations (the length of shortest paths is non-decreasing and each edge becomes saturated at most $O(n)$ times), and total time $O(nm^2)$.

What is the residual graph and why is it essential to the correctness of Ford-Fulkerson?

<showhide>
The residual graph $G_f$ has, for each edge $(u,v)$ with capacity $c(u,v)$ and flow $f(u,v)$: a forward edge $(u,v)$ with residual capacity $c(u,v) - f(u,v)$ (remaining room), and a backward edge $(v,u)$ with residual capacity $f(u,v)$ (flow that can be "undone"). Augmenting along backward edges allows the algorithm to "reroute" flow, which is necessary to reach the global optimum. Without backward edges, the algorithm might get stuck at a locally maximal but globally suboptimal flow.

## Applications of Max Flow

Max flow solves many problems: bipartite matching (unit-capacity flow), edge-disjoint paths (Menger's theorem), circulation, project selection, and more. The power comes from **reductions**: model the problem as a flow network, then apply a black-box max-flow algorithm.

**Bipartite matching**: given bipartite graph $G = (L \cup R, E)$, find maximum matching. Add source $s$ connected to all $l \in L$ and all $r \in R$ connected to sink $t$, all unit capacity. Max flow = max matching.

How does the max-flow min-cut theorem apply to bipartite matching via König's theorem?

<showhide>
König's theorem states that in a bipartite graph, the size of the maximum matching equals the size of the minimum vertex cover. This follows from max-flow min-cut: model the bipartite matching as a unit-capacity flow network. The min cut in this network (after removing the source and sink edges) corresponds exactly to the minimum vertex cover. The max-flow min-cut theorem then gives König's theorem as a corollary.
