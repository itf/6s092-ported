# Incremental Improvement: Matching

**Matching** in graphs is a fundamental combinatorial optimization problem. A **matching** $M$ in a graph $G = (V, E)$ is a set of edges with no shared vertices. A **maximum matching** has the most edges; a **perfect matching** covers all vertices. For bipartite graphs, the **Hopcroft-Karp algorithm** finds a maximum matching in $O(m\sqrt{n})$ time. For general graphs, **Edmonds' blossom algorithm** finds a maximum matching in $O(nm)$ (or $O(n^3)$ with more careful analysis).

The key concept is an **augmenting path**: a path from an unmatched vertex to another unmatched vertex that alternates between unmatched and matched edges. By the **Berge's theorem**, a matching $M$ is maximum iff there is no augmenting path. Each augmenting path increases $|M|$ by 1 (by toggling matched/unmatched along the path).

**Hopcroft-Karp** improves upon repeated single augmentation by finding **maximal sets of vertex-disjoint shortest augmenting paths** in each phase using BFS. Each phase takes $O(m)$ and increases matching size by at least 1; there are $O(\sqrt{n})$ phases because after $\sqrt{n}$ phases the shortest augmenting path has length $> \sqrt{n}$, and at most $\sqrt{n}$ more augmentations are needed.

## Berge's Theorem and Augmenting Paths

**Berge's theorem**: A matching $M$ is maximum if and only if $G$ contains no $M$-augmenting path.

Proof of "if": trivially, if there's an augmenting path, we can increase $|M|$ by 1.
Proof of "only if": suppose $M$ is not maximum; let $M^*$ be a maximum matching. Consider $M \oplus M^* = (M \setminus M^*) \cup (M^* \setminus M)$. The symmetric difference consists of alternating paths and cycles; the paths with more $M^*$ edges than $M$ edges are augmenting paths for $M$.

<showhide>
An **alternating path** starts at any vertex and alternates between unmatched and matched edges. An **augmenting path** is an alternating path that starts AND ends at unmatched vertices. Every augmenting path is alternating, but not vice versa. Augmenting paths are special because flipping matched/unmatched along them strictly increases the matching size.

## Matching in General Graphs: Blossoms

For general (non-bipartite) graphs, augmenting-path algorithms must handle **blossoms**: odd cycles contracted to a single vertex. Edmonds' blossom shrinking algorithm handles this: when BFS finds an odd cycle (blossom), contract it to a single vertex, find the augmenting path in the contracted graph, then expand it back.

The blossom algorithm runs in $O(n^3)$ (or $O(nm)$ with careful implementation). More modern algorithms (Micali-Vazirani) achieve $O(m \sqrt{n})$ for general graphs as well.

State Dilworth's theorem and explain how it relates to matching and antichain decomposition in partially ordered sets.

<showhide>
Dilworth's theorem states that in any finite partially ordered set (poset), the minimum number of chains needed to cover the poset equals the maximum size of an antichain. This is a direct consequence of König's theorem (bipartite min-cover = max-matching): model the poset as a bipartite graph where the left side represents elements as "sources" and the right side as "sinks," with edges for comparabilities. A minimum chain cover corresponds to a minimum path cover in the DAG, which equals $n$ minus the maximum matching. Maximum antichain = $n$ minus maximum matching as well.
