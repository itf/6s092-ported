# Randomization: Skip Lists

A **skip list** is a randomized data structure that supports search, insert, and delete in $O(\log n)$ expected time and $O(n)$ expected space. It generalizes a sorted linked list by adding multiple levels of "express lanes" that allow binary-search-like jumps.

A skip list on $n$ elements maintains $O(\log n)$ levels. Level 0 is a sorted linked list of all elements. Each element at level $i$ is independently promoted to level $i+1$ with probability $1/2$. Searching starts at the top level and descends when the next element exceeds the target. The expected search cost is $O(\log n)$ because the expected number of nodes at level $i$ is $n/2^i$, and we traverse $O(1)$ nodes per level.

Skip lists offer a practical alternative to balanced BSTs: they are simpler to implement and provide $O(\log n)$ expected performance for all dictionary operations. The expected space is $\sum_{i=0}^{\infty} n/2^i = 2n = O(n)$.

## Search and Expected Analysis

The **search** algorithm for target $x$: start at the top-left sentinel; at each position, if the next element at this level is $\leq x$, move right; otherwise drop down one level. Return when dropping below level 0.

The expected cost analysis uses a **backwards argument**: trace the search path in reverse (going up and left). The probability of going up at each step is $1/2$ (the node was promoted). The number of steps before reaching level $\lceil \log n \rceil$ is a negative binomial random variable with expected value $O(\log n)$.

<checkyourself>
Why does the backwards analysis of skip list search give a cleaner proof of the $O(\log n)$ expected bound?

<showhide>
In the forward direction, the structure seen during the search depends on the random choices, making analysis complex. The backwards analysis traces the search path in reverse from the found position, where each step is a coin flip: go up (if the node was promoted) with probability $1/2$, or go left with probability $1/2$. This gives a clean Markov chain analysis. The expected number of steps to reach height $\log n$ from height 0 is $O(\log n)$.

## Insert and Delete

**Insert** of element $x$: search for $x$'s position, insert at level 0, then flip a coin repeatedly—promote to level $i+1$ with probability $1/2$ until tails or the max level is reached. Each insertion takes $O(\log n)$ expected time (same as search).

**Delete** of element $x$: search for $x$ (tracking the predecessor at each level), then remove $x$'s node from each level it appears in. Also $O(\log n)$ expected time.

The maximum level is capped at $\lceil \log_2 n \rceil$ to avoid unbounded heights; the probability that any element exceeds this height is at most $1/n$.

How do skip lists compare to balanced BSTs (like red-black trees) in terms of worst-case and expected-case performance?

<showhide>
Red-black trees guarantee $O(\log n)$ worst-case for all operations. Skip lists give $O(\log n)$ expected time, but have $O(n)$ worst case (though this is exponentially unlikely). Skip lists are often preferred in practice for their simplicity: no complex rotations or rebalancing, easy concurrent implementations (lock-free versions exist), and good cache performance.
