# Divide & Conquer: Convex Hull, Median Finding

Divide and conquer is one of the most powerful algorithm design paradigms. A problem of size $n$ is split into subproblems of size $n/b$, each solved recursively, and then combined. The **Master Theorem** gives the runtime: if $T(n) = aT(n/b) + f(n)$, then $T(n) = \Theta(n^{\log_b a})$ when $f(n) = O(n^{\log_b a - \epsilon})$, $T(n) = \Theta(n^{\log_b a} \log n)$ when $f(n) = \Theta(n^{\log_b a})$, and $T(n) = \Theta(f(n))$ when $f(n) = \Omega(n^{\log_b a + \epsilon})$.

**Convex hull** asks: given $n$ points in the plane, find the smallest convex polygon containing all of them. The divide-and-conquer algorithm splits points by $x$-coordinate, recursively finds convex hulls of each half, then merges by finding upper and lower tangent lines. The merge step takes $O(n)$ using a two-pointer technique on sorted hulls, giving total time $T(n) = 2T(n/2) + O(n) = O(n \log n)$.

**Median finding** (selection in linear time) finds the $k$-th smallest element in an unsorted array. The randomized algorithm achieves $O(n)$ expected time. The deterministic median-of-medians algorithm achieves $O(n)$ worst-case time by choosing a pivot guaranteed to eliminate a constant fraction of elements at each step.

## Convex Hull via Divide and Conquer

The divide-and-conquer convex hull algorithm proceeds as follows: sort points by $x$-coordinate ($O(n \log n)$ once), split into left and right halves, recursively compute $\text{CH}(\text{left})$ and $\text{CH}(\text{right})$, then merge by finding the upper tangent (the line touching both hulls from above) and the lower tangent (from below). Each tangent can be found in $O(n)$ time by walking along the hull vertices.

The recurrence is $T(n) = 2T(n/2) + O(n)$, which by the Master Theorem gives $T(n) = O(n \log n)$. This matches the $\Omega(n \log n)$ lower bound for convex hull in the comparison model.

## Median Finding (Linear Selection)

The **median-of-medians** algorithm finds the $k$-th smallest element in $O(n)$ worst-case time:
1. Divide the $n$ elements into groups of 5; find the median of each group.
2. Recursively find the median $p$ of the $\lceil n/5 \rceil$ group medians.
3. Partition around $p$; recurse on the appropriate side.

The key insight: $p$ is guaranteed to be larger than at least $3\lfloor n/10 \rfloor$ elements and smaller than at least $3\lfloor n/10 \rfloor$ elements, so each recursive call has size at most $7n/10 + 6$. This gives $T(n) = T(n/5) + T(7n/10) + O(n) = O(n)$.
