# Randomization: Matrix Multiply, Quicksort

**Randomized algorithms** use random choices to achieve good expected performance or to simplify analysis. This lecture covers two fundamental randomized algorithms: **Freivalds' algorithm** for matrix multiplication verification and **randomized quicksort**.

**Freivalds' algorithm** verifies $A \cdot B = C$ for $n \times n$ matrices in $O(n^2)$ time with one-sided error $\leq 1/2$: pick a random vector $r \in \{0,1\}^n$, compute $A(Br)$ in $O(n^2)$, and check if it equals $Cr$. If $AB = C$ the check always passes; if $AB \neq C$ it fails with probability $\leq 1/2$. Repeating $k$ times reduces the error to $2^{-k}$.

**Randomized quicksort** picks a uniformly random pivot at each step. The expected number of comparisons is $O(n \log n)$: for elements $z_i$ and $z_j$ (with $i < j$), the probability that they are compared equals $\frac{2}{j - i + 1}$ (they are compared iff one of them is chosen as pivot before any element strictly between them). Summing over all pairs gives $\sum_{i < j} \frac{2}{j-i+1} = O(n \log n)$.

## Freivalds' Algorithm

Let $A, B, C$ be $n \times n$ matrices. We want to test $AB \stackrel{?}{=} C$. Freivalds' test: choose $r \in \{0,1\}^n$ uniformly at random, compute $x = Br$, $y = Ax$, $z = Cr$, and check $y = z$. Each matrix-vector product costs $O(n^2)$, so the total cost is $O(n^2)$.

**Correctness**: If $AB = C$, then $ABr = Cr$ always (no false negatives). If $AB \neq C$, let $D = AB - C \neq 0$. We need $\Pr[Dr = 0]$. By the Schwartz-Zippel lemma, $\Pr[Dr = 0] \leq 1/2$ for uniformly random $r \in \{0,1\}^n$.

Is Freivalds' algorithm a Las Vegas or Monte Carlo algorithm? Why?

<showhide>
Freivalds' is a Monte Carlo algorithm: it always runs in $O(n^2)$ time, but may return an incorrect answer (a false positive: saying $AB = C$ when it doesn't). There are no false negatives. A Las Vegas algorithm always returns the correct answer but has a random running time (e.g., randomized quicksort).

## Randomized Quicksort Analysis

Let $z_1 < z_2 < \cdots < z_n$ be the sorted order of the input. Define $X_{ij} = 1$ if $z_i$ and $z_j$ are ever compared. The total comparisons $= \sum_{i < j} X_{ij}$. 

Key claim: $\Pr[X_{ij} = 1] = \frac{2}{j - i + 1}$. Proof: $z_i$ and $z_j$ are compared iff one of them is the first pivot chosen from $\{z_i, z_{i+1}, \ldots, z_j\}$ (a set of $j-i+1$ elements). By symmetry, this has probability $\frac{2}{j-i+1}$.

Expected comparisons $= \sum_{i=1}^{n} \sum_{j=i+1}^{n} \frac{2}{j-i+1} = \sum_{i=1}^{n} \sum_{d=1}^{n-i} \frac{2}{d+1} \leq 2n \sum_{d=1}^{n} \frac{1}{d} = 2n H_n = O(n \log n)$.

What is the worst-case time complexity of randomized quicksort, and how does it differ from the expected complexity?

<showhide>
The worst-case is $O(n^2)$, which occurs (with probability $2/n!$) if the pivots are always the minimum or maximum element. However, the expected complexity is $O(n \log n)$, and the probability of $\Omega(n \log n \cdot c)$ comparisons for large $c$ decreases exponentially. In practice, randomized quicksort is very fast and widely used.
