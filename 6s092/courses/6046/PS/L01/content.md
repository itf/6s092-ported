# Overview, Interval Scheduling

This lecture introduces the course and motivates algorithm design through the lens of **interval scheduling**. Given a set of $n$ requests, each with a start time $s_i$ and finish time $f_i$, the goal is to select the maximum number of mutually compatible requests (no two overlap). This is a classic greedy problem that sets the stage for the course's themes of correctness, efficiency, and analysis.

The **greedy algorithm** for interval scheduling repeatedly selects the request with the earliest finish time that is compatible with all previously selected requests. This runs in $O(n \log n)$ time (dominated by sorting) and produces an optimal solution. The proof of optimality uses an exchange argument: any solution can be transformed into the greedy solution without decreasing the number of selected intervals.

Beyond interval scheduling, this lecture previews the major algorithmic paradigms covered in 6.046J: divide and conquer, dynamic programming, greedy algorithms, network flow, linear programming, and complexity theory. A recurring theme is that the choice of algorithm design technique is guided by the structure of the problem.

## Interval Scheduling

The interval scheduling problem is defined formally: given intervals $[s_i, f_i)$ for $i = 1, \ldots, n$, find a maximum-cardinality subset $S$ such that no two intervals in $S$ overlap. Two intervals $i$ and $j$ are **compatible** if $f_i \leq s_j$ or $f_j \leq s_i$.

The greedy algorithm sorts intervals by finish time and iterates greedily. Let $A$ be the greedy solution and $O$ any optimal solution. An exchange argument shows $|A| \geq |O|$: index the intervals in $A$ as $a_1, a_2, \ldots, a_k$ and those in $O$ as $o_1, o_2, \ldots, o_m$. By induction, $f(a_i) \leq f(o_i)$ for all $i$, so $k \geq m$.

The key invariant is that after each greedy choice, the remaining feasible set is at least as large as for any other choice with the same number of selections so far.

## Greedy Correctness via Exchange Argument

The exchange argument is a standard proof technique for greedy algorithms. Suppose an optimal solution $O$ differs from the greedy solution $A$. Find the first position $i$ where they differ; swap $o_i$ for $a_i$ in $O$. Show the resulting solution is still feasible and no worse. Repeat until $O = A$.

For interval scheduling, because $f(a_i) \leq f(o_i)$, replacing $o_i$ with $a_i$ only makes future constraints easier to satisfy. This inductive argument establishes that the greedy solution is optimal.

What is the difference between interval scheduling (maximize number of intervals) and interval partitioning (minimize number of resources/machines needed to schedule all intervals)?

<showhide>
Interval scheduling selects a maximum subset of mutually compatible intervals using one resource. Interval partitioning assigns all intervals to the minimum number of resources (machines) such that no resource has two overlapping intervals. The greedy for partitioning assigns each interval to any compatible resource, opening a new one if needed; the minimum number of resources equals the maximum depth (maximum overlap) at any point in time.
