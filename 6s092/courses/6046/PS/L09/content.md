# Augmentation: Range Trees

**Data structure augmentation** enriches an existing structure with extra information to support new operations efficiently. This lecture focuses on **range trees**, which answer orthogonal range queries: given $n$ points in $\mathbb{R}^d$, report all points in a $d$-dimensional axis-aligned box $[x_1, x_2] \times [y_1, y_2] \times \cdots$.

A **1D range tree** is simply a sorted array or balanced BST. A query $[a, b]$ finds all points in $O(\log n + k)$ time, where $k$ is the output size: locate $a$ and $b$ in $O(\log n)$, then enumerate. For 2D, a **2D range tree** augments each node of the $x$-BST with a **fractional cascading** structure or a secondary $y$-BST on the points in that subtree.

Without fractional cascading, a 2D range tree answers queries in $O(\log^2 n + k)$ time and uses $O(n \log n)$ space. With **fractional cascading**, the time drops to $O(\log n + k)$ by propagating pointers between secondary structures to avoid re-searching from scratch at each node.

## 1D Range Trees

A 1D range tree stores $n$ points in a balanced BST sorted by coordinate. A **range query** $[a, b]$ proceeds by finding the split node (the deepest node where the query range branches), then collecting all points in $[a, b]$ from the left and right subtrees.

The query takes $O(\log n + k)$ time: $O(\log n)$ to find the boundary nodes, then $O(k)$ to output $k$ results. Insertion and deletion take $O(\log n)$ time (maintaining balance).

## Fractional Cascading

**Fractional cascading** reduces the $\log^2 n$ factor in 2D range trees to $\log n$. The key idea: augment each node $v$'s secondary $y$-list with every other element from its children's $y$-lists. This allows the $y$-search at child nodes to start from the found position in the parent, using a precomputed pointer, rather than re-searching.

More precisely: for node $v$ with children $v_L$ and $v_R$, the augmented list $C[v]$ merges $C[v_L]$ (half) and $C[v_R]$ (half) into the $y$-sorted list, with pointers. Binary searching $C[v]$ once costs $O(\log n)$, and descending to children costs $O(1)$ per node thereafter.

How would you extend a 2D range tree to handle 3D orthogonal range queries, and what would the query time be?

<showhide>
Build a primary BST on the first coordinate; augment each node with a 2D range tree on the second and third coordinates of the points in its subtree. This gives $O(\log^3 n + k)$ query time and $O(n \log^2 n)$ space. In general, $d$-dimensional range trees achieve $O(\log^d n + k)$ query time and $O(n \log^{d-1} n)$ space. Fractional cascading can reduce the time to $O(\log^{d-1} n + k)$ for $d \geq 2$.
