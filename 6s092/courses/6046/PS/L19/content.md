# Synchronous Distributed Algorithms: Symmetry-Breaking, Shortest-Paths Spanning Trees

**Distributed algorithms** operate on a network of processors (nodes) that communicate by passing messages. In the **synchronous model**, all processors execute in lockstep rounds: in each round, each processor sends messages, receives messages sent in the same round, and updates its local state. This lecture covers two fundamental problems: **leader election / symmetry-breaking** and **computing BFS spanning trees**.

In a **ring network** of $n$ nodes, the **symmetry-breaking** problem asks nodes to elect a unique leader. If all nodes are identical (no IDs), this is impossible by symmetry. With unique IDs, the **LCR algorithm** (Le Lann, Chang, Roberts) elects the maximum-ID node as leader in $O(n)$ rounds using $O(n^2)$ messages: each node forwards IDs clockwise, discarding IDs smaller than what it has seen, and the node whose ID travels all the way around becomes the leader.

**BFS spanning tree**: given a source $s$, all nodes compute the BFS tree rooted at $s$ in $O(D)$ rounds (where $D$ is the diameter), using $O(m)$ messages. Each node joins the tree when it first receives a message from $s$.

## Leader Election in Rings

In a synchronous ring of $n$ nodes with unique IDs, the LCR algorithm works as follows: initially, each node sends its ID. In each subsequent round, each node receives an ID from its left neighbor, forwards it to the right if it is larger than any ID seen so far, and discards it otherwise. When a node receives its own ID back, it declares itself the leader and broadcasts this.

**Message complexity**: In the worst case (IDs sorted in increasing order around the ring), the maximum ID travels $n$ hops, the second-largest travels $n$ hops (until meeting the max), etc. Total messages $= O(n^2)$ worst case, $O(n \log n)$ on average.

Why is leader election impossible in a synchronous ring of identical processes (no unique IDs)?

<showhide>
By symmetry: if all processes are identical and start in the same state, they all behave identically in every round. If one process elected itself leader, all processes would elect themselves, violating the requirement of a unique leader. Formally, this is proved by a **symmetry argument** or by showing the execution is invariant under rotation, so no process can distinguish itself from others.

## BFS Spanning Trees in General Networks

Given a designated source $s$, computing a BFS spanning tree in a synchronous network takes exactly $D$ rounds (where $D$ = eccentricity of $s$ = longest shortest path from $s$). In each round $i$: the nodes at distance $i-1$ from $s$ send "JOIN" messages to all their neighbors; any neighbor that hasn't joined yet joins the BFS tree (designating the first sender as parent) and sends JOIN messages in round $i+1$.

**Message complexity**: $O(m)$ messages total—each edge is processed at most twice (once in each direction). **Round complexity**: $O(D)$.

In a distributed algorithm, what does it mean for an algorithm to be "correct" versus "terminated"? How does a node know the computation is complete?

<showhide>
**Correctness** means all nodes compute the right output (e.g., the correct BFS tree edges). **Termination** means all nodes stop executing. In synchronous BFS, nodes know they are done when they receive no new messages in a round. To globally detect termination, additional mechanisms are needed: for example, nodes can propagate "convergecast" messages back to the source once their subtree is done, after $D$ more rounds. Without such a mechanism, nodes can't distinguish "no more messages will ever come" from "messages are still in transit."
