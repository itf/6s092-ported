# Complexity: P, NP, NP-completeness, Reductions

**Computational complexity theory** classifies problems by the resources (time, space) required to solve them. The class **P** contains problems solvable in deterministic polynomial time. The class **NP** contains problems where solutions can be verified in polynomial time (equivalently, solvable in non-deterministic polynomial time). The central open question is whether $\mathbf{P} = \mathbf{NP}$.

A problem $B$ is **NP-hard** if every problem in NP reduces to $B$ in polynomial time. A problem is **NP-complete** if it is both in NP and NP-hard. The first NP-complete problem was shown by **Cook's theorem** (1971): **SAT** (Boolean satisfiability) is NP-complete. **Karp's 21 problems** (1972) established NP-completeness of many combinatorial problems (3-SAT, Clique, Vertex Cover, Hamiltonian Cycle, etc.) via polynomial reductions.

**Polynomial-time reductions**: $A \leq_P B$ means "$A$ reduces to $B$ in polynomial time"—an efficient algorithm for $B$ yields one for $A$. To show $B$ is NP-hard, reduce a known NP-hard problem $A$ to $B$.

## P, NP, and the P vs. NP Question

$\mathbf{P} \subseteq \mathbf{NP}$: any problem with a polynomial-time algorithm also has a polynomial-time verifier (just run the algorithm and ignore the certificate). Whether $\mathbf{NP} \subseteq \mathbf{P}$ (i.e., $\mathbf{P} = \mathbf{NP}$) is unknown.

If $\mathbf{P} = \mathbf{NP}$: all NP-complete problems would be efficiently solvable, breaking most modern cryptography. The consensus view (but unproven) is $\mathbf{P} \neq \mathbf{NP}$.

## NP-Completeness Reductions

Classic NP-completeness reductions chain from SAT:

$\text{SAT} \to \text{3-SAT} \to \text{Independent Set} \to \text{Vertex Cover} \to \text{Set Cover}$

$\text{3-SAT} \to \text{Clique} \to \text{Hamiltonian Cycle} \to \text{TSP}$

Each arrow is a polynomial reduction. Understanding these reductions and their structure is key to both proving hardness and finding approximation algorithms.

What are the complexity classes co-NP, NP $\cap$ co-NP, and how do they relate to P?

<showhide>
**co-NP** is the class of problems whose complements are in NP (i.e., "no" instances have polynomial-time verifiable proofs). For example, TAUTOLOGY (is every assignment satisfying?) is in co-NP. We have P $\subseteq$ NP $\cap$ co-NP; it is believed (but unproven) that NP $\neq$ co-NP. Problems like graph isomorphism are in NP $\cap$ co-NP but not known to be in P. Integer factoring is in NP $\cap$ co-NP (and is the basis of RSA security).
</showhide>
