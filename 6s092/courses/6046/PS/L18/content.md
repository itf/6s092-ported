# Complexity: Fixed-Parameter Algorithms

**Fixed-parameter tractability (FPT)** provides a refined view of NP-hard problems by studying how complexity scales with a "parameter" $k$ that is often small in practice. A problem is **FPT** with respect to parameter $k$ if it can be solved in time $f(k) \cdot n^{O(1)}$ for some computable function $f$. The key: the exponential dependence is only in $k$, not $n$.

The FPT framework is formalized by the **W-hierarchy**: $\text{FPT} \subseteq W[1] \subseteq W[2] \subseteq \cdots \subseteq \text{XP}$. Most natural parameterizations of NP-hard problems are either FPT or W[1]-hard (analogous to NP-hard in parameterized complexity). A problem is **W[1]-hard** if an FPT algorithm for it would imply FPT for all W[1] problems (conjectured impossible).

Classic FPT algorithms use **bounded search trees**, **kernelization**, and **color-coding**. For example, **$k$-Vertex Cover** (does a graph have a vertex cover of size $\leq k$?) is FPT: a $O(2^k \cdot (n + m))$ algorithm exists via branching, and a $O(kn + k^2)$-time kernelization reduces any instance to a kernel of size $O(k^2)$.

## Kernelization

**Kernelization**: a polynomial-time preprocessing that reduces any instance $(I, k)$ to an equivalent instance $(I', k')$ with $|I'| \leq g(k)$ and $k' \leq k$ for some function $g$. The kernel $I'$ has size bounded only in $k$, not $n$.

For $k$-Vertex Cover: the **crown reduction** and **LP-based reduction** reduce the graph to at most $2k$ vertices (the $2k$-kernel). Any vertex of degree $> k$ must be in the cover (include it, reduce $k$ by 1). After this, if more than $k^2$ edges remain, answer "NO" (since $k$ vertices cover at most $k^2$ edges).

What is the difference between a parameterized problem being FPT versus being in XP?

<showhide>
**FPT**: solvable in $f(k) \cdot n^{O(1)}$ time—polynomial in $n$ for any fixed $k$, with the parameter controlling only the constant/prefactor. **XP** (slice-wise polynomial): solvable in $n^{f(k)}$ time—still polynomial for fixed $k$, but the exponent of $n$ depends on $k$. XP is much larger than FPT; for example, $k$-Clique is in XP ($O(n^k)$) but is W[1]-hard (not FPT unless W[1] = FPT). FPT algorithms are practical for small $k$; XP algorithms become impractical quickly.

## Color-Coding

**Color-coding** (Alon, Yuster, Zwick 1995) is a technique for finding paths or subgraphs of size $k$ in graphs. Color each vertex randomly with one of $k$ colors. A path $v_1, v_2, \ldots, v_k$ is **colorful** if all vertices get distinct colors. If a $k$-path exists, a random coloring makes it colorful with probability $k!/k^k \geq e^{-k}$. Then find a colorful $k$-path by DP in $O(2^k \cdot nm)$.

By derandomization (using $(n, k)$-perfect hash families of size $e^k \cdot k^{O(\log k)} \cdot \log n$), the algorithm is made deterministic. The total time is $e^k \cdot k^{O(\log k)} \cdot nm \cdot \log n = O^*(e^k)$.

What is treewidth, and why does it play a central role in parameterized algorithms?

<showhide>
**Treewidth** measures how "tree-like" a graph is. A tree has treewidth 1; the complete graph $K_n$ has treewidth $n-1$. Many NP-hard problems (vertex cover, independent set, Hamiltonian cycle, graph coloring) become polynomial-time solvable on graphs of bounded treewidth via dynamic programming on the tree decomposition. The DP runs in $f(k) \cdot n$ time where $k$ is the treewidth, making these problems FPT in treewidth.
</showhide>
