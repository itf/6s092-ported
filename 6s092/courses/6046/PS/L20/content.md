# Asynchronous Distributed Algorithms: Shortest-Paths Spanning Trees

The **asynchronous distributed model** removes the synchrony assumption: messages can be delayed arbitrarily (but are eventually delivered), and processors run at independent speeds. This makes algorithm design much harder—algorithms must be correct regardless of message delivery order.

In the **asynchronous model**, the key difficulty is that a process cannot distinguish between "no message has been sent to me" and "a message is in transit." Classic problems like BFS tree construction require new techniques: **Bellman-Ford-style relaxation** works correctly without synchronized rounds, at the cost of more messages and time.

The **asynchronous Bellman-Ford algorithm** computes single-source shortest paths. Each node maintains an estimate $d[v]$ of its distance from the source, initialized to $\infty$ (except $d[s] = 0$). Whenever $d[v]$ decreases, $v$ sends updated distance to all neighbors; neighbor $u$ updates $d[u] = \min(d[u], d[v] + w(v,u))$. This converges to correct shortest paths but may use $O(nm)$ messages in the worst case.

## Asynchronous BFS vs. Bellman-Ford

The synchronous BFS algorithm fails asynchronously because a node might receive messages from different "waves" out of order and incorrectly identify its parent. The Bellman-Ford approach is robust: nodes always update to the minimum distance seen, regardless of message order.

**Convergence time**: In an asynchronous network where message delays are at most $d$ time units, Bellman-Ford converges in at most $O(n \cdot d)$ time (at most $n-1$ relaxation rounds, each taking at most $d$ time). The number of messages is $O(nm)$ worst case (each node can be relaxed $n-1$ times, sending $\deg(v)$ messages each time).

Why does the synchronous BFS algorithm fail in an asynchronous network?

<showhide>
In synchronous BFS, a node joins the BFS tree from the first message it receives, which is guaranteed to be from distance-1 nodes in round 1, distance-2 in round 2, etc. Asynchronously, messages may arrive out of order: a message from a distance-3 node might arrive before a message from a distance-2 node (due to different delays). If a node adopts the wrong parent, its distance estimate is incorrect and it may propagate errors. Bellman-Ford avoids this by always keeping the minimum estimate, regardless of message order.

## Asynchronous Leader Election

In an asynchronous ring with unique IDs, the **Hirschberg-Sinclair algorithm** elects a leader in $O(n \log n)$ messages (optimal up to constants) using a "doubling" strategy: nodes compete in stages of increasing power, where at stage $k$, nodes try to "dominate" their $2^k$-neighborhood.

The key insight: in each stage, only half (approximately) of the surviving candidates advance, so there are $O(\log n)$ stages. Each stage uses $O(n)$ messages (the candidates probe linearly growing neighborhoods). Total: $O(n \log n)$ messages.

What is the key challenge of distributed computing that makes asynchronous algorithms harder than synchronous ones, beyond just message ordering?

<showhide>
Beyond message ordering, the fundamental challenge is **impossibility of global synchronization**: in an asynchronous system, there is no shared clock, and a process cannot tell whether silence from a neighbor means the neighbor is slow, crashed, or simply has nothing to send. This leads to the **FLP impossibility result** (Fischer, Lynch, Paterson 1985): in an asynchronous network where even one process can fail by crashing, it is impossible to achieve consensus (agreement) deterministically. This is why fault-tolerant distributed algorithms require careful design.
</showhide>
