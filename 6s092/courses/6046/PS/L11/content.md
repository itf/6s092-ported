# Dynamic Programming: All-Pairs Shortest Paths

**All-pairs shortest paths (APSP)** computes the shortest path between every pair of vertices in a weighted graph. For a graph $G = (V, E)$ with $|V| = n$ and $|E| = m$, three key algorithms are: **Floyd-Warshall** ($O(n^3)$, handles negative weights), **Johnson's algorithm** ($O(nm + n^2 \log n)$, suitable for sparse graphs), and repeated **Bellman-Ford** ($O(n^2 m)$, for general graphs).

**Floyd-Warshall** uses DP on the set of intermediate vertices. Let $d^{(k)}[i][j]$ = shortest path from $i$ to $j$ using only vertices $\{1, \ldots, k\}$ as intermediates. Recurrence: $d^{(k)}[i][j] = \min(d^{(k-1)}[i][j],\ d^{(k-1)}[i][k] + d^{(k-1)}[k][j])$. Initialize $d^{(0)}[i][j] = w(i,j)$ (or $\infty$ if no edge). After $n$ iterations, $d^{(n)}$ holds the true shortest paths.

**Johnson's algorithm** uses **reweighting** to eliminate negative edges, then runs Dijkstra from each vertex. Reweighting: add a virtual vertex $s$, run Bellman-Ford from $s$ to get potentials $h[v]$, redefine $\hat{w}(u,v) = w(u,v) + h[u] - h[v] \geq 0$. This makes all edge weights non-negative while preserving shortest-path structure.

## Floyd-Warshall Algorithm

Floyd-Warshall runs in $O(n^3)$ time and $O(n^2)$ space (with in-place updates). It handles negative edge weights and detects negative cycles: a negative cycle exists iff any $d[i][i] < 0$ at the end.

The algorithm can also compute the **transitive closure** of a graph: replace $\min/+$ with $\lor/\land$ (OR/AND), setting $d^{(0)}[i][j] = 1$ if edge $(i,j)$ exists, else 0.

<checkyourself>
Why does Floyd-Warshall work correctly when edges have negative weights (unlike Dijkstra), and what condition would cause it to give wrong answers?

<showhide>
Floyd-Warshall is based on DP over intermediate vertices, not on a greedy relaxation order. It considers all possible intermediate vertices, so negative edges don't cause problems. However, it gives wrong answers (and may fail to terminate) if there are negative-weight cycles, because the "shortest path" through a negative cycle has cost $-\infty$. The algorithm can detect negative cycles by checking if any diagonal entry $d[i][i] < 0$.

## Johnson's Algorithm

Johnson's algorithm runs APSP on sparse graphs faster than Floyd-Warshall. Steps:
1. Add a new vertex $s$; add zero-weight edges from $s$ to all other vertices.
2. Run Bellman-Ford from $s$ to get potentials $h[v]$ = shortest-path distance from $s$ to $v$.
3. Reweight: $\hat{w}(u,v) = w(u,v) + h[u] - h[v] \geq 0$.
4. Run Dijkstra from each of the $n$ vertices using $\hat{w}$.
5. Recover true distances: $d(u,v) = \hat{d}(u,v) - h[u] + h[v]$.

Total time: $O(nm + n^2 \log n)$ with a Fibonacci heap (or $O(nm \log n)$ with a binary heap).

For what types of graphs is Johnson's algorithm faster than Floyd-Warshall, and when is Floyd-Warshall preferable?

<showhide>
Johnson's is faster for **sparse** graphs where $m = O(n^{1+\epsilon})$ for $\epsilon < 1$: then $O(nm + n^2 \log n) = o(n^3)$. For **dense** graphs ($m = \Theta(n^2)$), both are $O(n^3)$ (or $O(n^2 \log n)$ for Johnson's with Fibonacci heap, but the $nm$ term dominates). Floyd-Warshall has simpler code, handles dense graphs well, and has smaller constants in practice.
