# Greedy Algorithms: Minimum Spanning Tree

A **minimum spanning tree (MST)** of a weighted undirected graph $G = (V, E)$ is a spanning tree of minimum total edge weight. MSTs arise in network design, cluster analysis, and approximation algorithms. Three classical algorithms are **Kruskal's** ($O(m \log m)$), **Prim's** ($O(m + n \log n)$ with Fibonacci heap), and **Borůvka's** ($O(m \log n)$).

All MST algorithms are based on the **cut property** and **cycle property**:
- **Cut property**: For any cut $(S, V \setminus S)$, the minimum-weight crossing edge is in every MST (assuming unique edge weights).
- **Cycle property**: The maximum-weight edge on any cycle is in no MST.

These properties certify correctness of greedy MST algorithms. **Kruskal's algorithm** sorts edges by weight and adds each edge if it doesn't form a cycle (using a Union-Find/Disjoint Set Union structure). **Prim's algorithm** grows a single tree from a root by repeatedly adding the minimum-weight edge leaving the current tree.

## Kruskal's Algorithm

Kruskal's algorithm: sort $m$ edges by weight ($O(m \log m)$); initialize $n$ singleton components; for each edge $(u,v)$ in sorted order, if $u$ and $v$ are in different components, add $(u,v)$ to the MST and merge the components (Union-Find). Total time: $O(m \log m) = O(m \log n)$ (since $m \leq n^2$).

Correctness follows from the cut property: when edge $(u,v)$ is considered, it is the minimum-weight edge crossing the cut between $u$'s component and $v$'s component (all lighter edges have already been processed and either added to the MST or created a cycle within the same component).

<checkyourself>
Explain why Kruskal's algorithm is correct using the cut property.

<showhide>
When Kruskal's adds edge $(u,v)$, let $S$ = the component containing $u$. Edge $(u,v)$ is the lightest edge crossing the cut $(S, V\setminus S)$ at this point: any lighter edge has already been processed and either (a) added to the MST (but then $u$ and $v$ would already be connected, contradiction) or (b) discarded because it formed a cycle within one component (so it doesn't cross this cut). By the cut property, the minimum crossing edge is in every MST, so Kruskal's is correct.

## Prim's Algorithm

Prim's algorithm grows an MST from a single root vertex $r$: maintain a min-priority queue of vertices not in the MST, keyed by the minimum edge weight connecting them to the current tree. At each step, extract the minimum-key vertex $u$, add it to the MST, and update keys of its neighbors.

With a Fibonacci heap, decrease-key costs $O(1)$ amortized, giving total time $O(m + n \log n)$. With a binary heap, decrease-key costs $O(\log n)$, giving $O(m \log n)$.

What happens to MST algorithms when edge weights are not unique? Do the cut property and cycle property still hold?

<showhide>
When edge weights are not unique, the MST may not be unique (there can be multiple MSTs of the same weight). The cut property and cycle property still hold in a modified form: the minimum-weight edge crossing a cut is in *some* MST (not necessarily all). To break ties consistently, algorithms can use secondary sorting keys (e.g., edge index). Kruskal's and Prim's still produce a valid MST; they just might output different ones.
</showhide>
