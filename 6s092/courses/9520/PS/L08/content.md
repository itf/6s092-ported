# Multiclass Classification

Binary classifiers output $\hat{y} \in \{-1,+1\}$. **Multiclass classification** extends this to $k > 2$ classes. Strategies include **One-vs-All (OVA)**: train $k$ binary classifiers, each distinguishing class $j$ from all others; predict $\hat{y} = \arg\max_j f_j(x)$. Alternative schemes include **All-Pairs (OVO)**: train $\binom{k}{2}$ classifiers, each on a pair of classes, then use majority voting.

Rifkin and Klautau's analysis shows that OVA with well-calibrated classifiers (e.g., RLSC) performs as well as more complex multiclass formulations. The key insight: if each binary classifier $f_j$ estimates $P(y=j|x) - P(y\neq j|x)$, then $\arg\max_j f_j(x)$ gives the Bayes-optimal prediction. RLSC naturally estimates calibrated real-valued scores.

Native multiclass SVM formulations (Weston-Watkins, Crammer-Singer) jointly learn all $k$ weight vectors subject to constraints $w_{y_i} \cdot x_i - w_j \cdot x_i \geq 1 - \xi_i^j$ for all $j \neq y_i$. These are theoretically appealing but computationally more demanding and empirically similar to OVA.

## One-vs-All Strategy

<checkyourself>
In OVA with $k$ classes, how many binary classifiers are trained? Compare the training complexity to All-Pairs (OVO).

<showhide>
OVA trains $k$ binary classifiers (one per class). OVO trains $\binom{k}{2} = k(k-1)/2$ classifiers (one per pair). For large $k$, OVO is much more expensive: e.g., $k=100$ classes gives 100 vs 4950 classifiers. However, each OVO classifier is trained on only $2n/k$ points (two classes), so individual classifiers are faster to train. OVA is generally preferred for large $k$.

<checkyourself>
Explain why RLSC is argued to be better suited for OVA than a classifier that only outputs $\pm 1$ decisions.

<showhide>
RLSC outputs real-valued scores $f_j(x) \in \mathbb{R}$, not just $\pm 1$. These scores approximate the conditional class probability difference $P(y=j|x) - P(y\neq j|x)$. Taking $\arg\max_j f_j(x)$ exploits the calibration of the scores. A classifier outputting only $\pm 1$ loses this calibration: ties and close decisions cannot be broken by confidence. Rifkin and Klautau show that well-calibrated OVA is Bayes-consistent.
</showhide>
