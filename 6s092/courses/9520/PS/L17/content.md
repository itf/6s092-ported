# Active Learning

In **active learning**, the learner can query labels for chosen unlabeled examples, rather than receiving random labeled data. The goal is to achieve low generalization error using far fewer labeled examples than passive learning requires. The learner adaptively selects the most informative queries.

The **pool-based active learning** setting: a pool of $n$ unlabeled examples is available; the algorithm selects which to label. The **query by committee (QBC)** strategy maintains a version space $\mathcal{V}$ (the set of hypotheses consistent with all labeled data), and queries the example where the committee of hypotheses in $\mathcal{V}$ disagrees most. This halves the version space volume at each step, giving $O(\log(1/\epsilon))$ query complexity for simple classes.

For SVM-based active learning, **uncertainty sampling** queries the example closest to the current decision boundary ($x^* = \arg\min_x |f(x)|$), where the model is least confident. For regression in RKHS, **optimal experimental design** queries the point that maximally reduces predictive variance. Active learning can achieve exponential savings: $O(\log(1/\epsilon))$ labels instead of $O(1/\epsilon^2)$ for learning a threshold in 1D.

## Version Space and Query Complexity

<checkyourself>
Define the version space and explain how QBC (Query by Committee) uses it to select queries.

<showhide>
The version space $\mathcal{V}$ is the set of all hypotheses in $\mathcal{H}$ consistent with the labeled examples seen so far. QBC maintains a random sample from $\mathcal{V}$ (the "committee") and queries the unlabeled example with the highest **vote entropy** — the example where the committee is most divided. This tends to query near the decision boundary of the true hypothesis, maximally reducing the version space volume.

Give one setting where active learning provides an exponential (in $1/\epsilon$) improvement over passive learning, and one setting where it provides only a polynomial improvement.

<showhide>
**Exponential improvement:** Threshold functions in 1D (or halfspaces in $d$ dimensions with specific distributions). Binary search queries achieve $O(\log(1/\epsilon))$ labels vs passive $O(1/\epsilon^2)$. **Polynomial improvement:** For general halfspaces under arbitrary distributions, the improvement is only polynomial (e.g., $O(d\log(1/\epsilon)/\epsilon)$ vs $O(d/\epsilon^2)$). The savings depend strongly on the geometry of the decision boundary and the data distribution.
