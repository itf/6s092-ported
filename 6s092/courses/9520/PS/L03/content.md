# Reproducing Kernel Hilbert Spaces

A **Reproducing Kernel Hilbert Space** (RKHS) $\mathcal{H}_K$ is a Hilbert space of functions $f: \mathcal{X} \to \mathbb{R}$ associated with a symmetric positive semi-definite kernel $K: \mathcal{X} \times \mathcal{X} \to \mathbb{R}$, characterized by the **reproducing property**: $f(x) = \langle f, K(\cdot, x) \rangle_{\mathcal{H}_K}$ for all $f \in \mathcal{H}_K$ and $x \in \mathcal{X}$.

The Mercer theorem guarantees that any continuous, symmetric, positive semi-definite kernel $K$ on a compact domain has an eigenexpansion $K(x,t) = \sum_{i=1}^\infty \mu_i \phi_i(x)\phi_i(t)$, where $\mu_i \geq 0$ and $\{\phi_i\}$ are orthonormal eigenfunctions in $L^2$. The RKHS norm is then $\|f\|^2_{\mathcal{H}_K} = \sum_i f_i^2/\mu_i$ where $f_i = \langle f, \phi_i \rangle_{L^2}$.

The **Representer Theorem** is the key computational result: the minimizer of any Tikhonov functional $\min_{f \in \mathcal{H}_K}\left[\frac{1}{n}\sum_i \ell(f(x_i), y_i) + \lambda \|f\|^2_{\mathcal{H}_K}\right]$ always has the form $f^*(x) = \sum_{i=1}^n c_i K(x, x_i)$, reducing an infinite-dimensional optimization to a finite-dimensional one.

## Kernels and the Reproducing Property

State the reproducing property and explain why it is useful for learning algorithms.

<showhide>
The reproducing property says $f(x) = \langle f, K(\cdot, x)\rangle_{\mathcal{H}_K}$. This is useful because it allows function evaluations to be expressed as inner products in the Hilbert space, which means all computations involving $f$ only require kernel evaluations $K(x_i, x_j)$ — never explicit feature maps. This is the "kernel trick."

Give an example of a kernel that corresponds to a finite-dimensional feature space and explain how this simplifies computation.

<showhide>
The polynomial kernel $K(x,t) = (x \cdot t + 1)^p$ corresponds to a feature map $\phi(x)$ containing all monomials of degree $\leq p$. The feature space is finite-dimensional (dimension $\binom{d+p}{p}$), so we could explicitly compute feature vectors. However, evaluating $K(x,t)$ directly costs $O(d)$ instead of $O(d^p)$, illustrating the computational advantage of kernels even in the finite-dimensional case.
